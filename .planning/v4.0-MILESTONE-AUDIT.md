### Integration Gap Details

**INT-01: CategoryDrillDown missing viewMode**

- **Files:** `components/money/budget-overview.tsx` → `components/money/category-drill-down.tsx`
- **Impact:** When user selects Household tab and clicks a budget category, drill-down shows personal transactions instead of household
- **Fix:** Add `view` prop to CategoryDrillDown, pass `viewMode` from BudgetOverview
- **Requirements affected:** HOUS-03, HOUS-07, BUDG-05

**INT-02: useSpendingTrends missing viewMode**

- **Files:** `components/money/budget-overview.tsx` calls `useSpendingTrends(12)` without viewMode
- **Impact:** Spending trend chart always shows personal data regardless of Mine/Household tab
- **Fix:** Change to `useSpendingTrends(12, viewMode)` — hook already accepts optional second parameter
- **Requirements affected:** HOUS-03, HOUS-07, BUDG-03
