## Critical Integration Breaks

### Break 1: `title` missing from autosave POST payload

- **From:** `components/journal/journal-entry-modal.tsx` → `scheduleSave({ content, mood, word_count, prompt_key })`
- **To:** `POST /api/journal` → `journalEntryFormSchema` requires `title: z.string().trim().min(1)`
- **Impact:** Every new entry creation returns HTTP 400 — no entries can be created
- **Affected:** ENTR-01, ENTR-04, ENTR-05 (cascading: PRMT-01/02 prompt saves also fail)
- **Fix:** Make `title` optional in schema (`.optional().default("")`) OR add title input to modal

### Break 2: `mood: null` rejected by Zod schema

- **From:** `journal-entry-modal.tsx` → `useState<number | null>(null)` → `scheduleSave({ mood: null })`
- **To:** `journalEntryFormSchema` → `mood: z.number().int().min(1).max(5).default(3)` (`.default()` only replaces `undefined`, not `null`)
- **Impact:** Autosave fails before user selects mood (POST and PATCH both reject `null`)
- **Affected:** ENTR-01, ENTR-02, ENTR-04
- **Fix:** Add `.nullable()` to mood field OR strip null from payload before sending
