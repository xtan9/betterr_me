## Unsatisfied Requirements Detail

### TXNS-07: CSV Import (Phase 20)

- **Description:** User can import transactions via CSV file upload with column mapping and duplicate detection
- **Status:** Not implemented
- **Evidence:** No CSV upload endpoint, no file parser, no column mapping UI, no duplicate detection logic exists anywhere in codebase
- **Impact:** Users with bank accounts not supported by Plaid have no bulk import option

### MGMT-01: CSV Export (Phase 25)

- **Description:** User can export transactions as CSV with selectable date range
- **Evidence:** No export endpoint, no CSV generation code, no download UI
- **Impact:** Users cannot extract their transaction data

### MGMT-02: Data Deletion (Phase 25)

- **Description:** User can delete all money data and household membership in a single action
- **Evidence:** No deletion endpoint, no confirmation flow, no cascade delete logic
- **Impact:** Users cannot remove their financial data from the system
