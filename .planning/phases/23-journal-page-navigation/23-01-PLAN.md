Create four new client components in `components/journal/`:

**1. `journal-mood-dot.tsx`** — Mood-to-color mapping + dot indicator:
- Export `MOOD_COLORS: Record<number, string>` mapping mood values 1-5 to Tailwind bg classes:
  - 5: `bg-green-500` (amazing), 4: `bg-emerald-400` (good), 3: `bg-yellow-400` (okay), 2: `bg-orange-400` (not great), 1: `bg-red-400` (awful)
- Export `moodDotColor(mood: number): string` returning the class or `bg-muted-foreground` for unknown values.
- Export `JournalMoodDot` component: tiny `div` with `size-1.5 rounded-full` + mood color class. Accept `mood: number` and optional `className` props.

**2. `journal-calendar.tsx`** — Calendar with mood-colored entry dots:
- `"use client"` component.
- Props: `onDayClick: (date: string) => void` — passes YYYY-MM-DD string for clicked date. `refreshKey?: number` — when changed, triggers a `mutate()` on the SWR hook to refetch calendar data (used by parent after modal close).
- State: `currentMonth` (Date) initialized to `new Date()`.
- Use `useJournalCalendar(year, month)` where year/month extracted from `currentMonth` state (`getFullYear()`, `getMonth() + 1`).
- Build a `useMemo` entry lookup `Map<string, JournalCalendarDay>` from the entries array for O(1) access.
- Import `Calendar` from `@/components/ui/calendar` and render it with:
  - `month={currentMonth}`, `onMonthChange={setCurrentMonth}`
  - `onDayClick={(day) => onDayClick(getLocalDateString(day))}` — ALWAYS use `getLocalDateString()` from `lib/utils`, never `.toISOString()`.
  - `disabled={{ after: new Date() }}` — prevent selecting future dates.
  - Custom `components={{ DayContent: JournalDayContent }}` where `JournalDayContent` is defined OUTSIDE the render function (memoize component reference, per Research Pitfall 2). Use a React context or ref to pass the entryMap to the DayContent component. The simplest approach: define a module-level context `EntryMapContext = createContext<Map<string, JournalCalendarDay>>(new Map())`, wrap Calendar in `<EntryMapContext.Provider value={entryMap}>`, and have `JournalDayContent` consume it via `useContext`.
  - `JournalDayContent` renders: default `<DayContent {...props} />` wrapped in a `div.relative.flex.flex-col.items-center`, with a `<JournalMoodDot>` absolutely positioned below (`absolute -bottom-1`) when the day has an entry. Import `DayContent` and `DayContentProps` from `react-day-picker`.
- Add a `useEffect` watching `refreshKey`: when it changes (and is truthy), call the hook's `mutate()` to refetch the calendar data. This allows the parent page to trigger a refresh after modal create/edit/delete by incrementing `refreshKey`.
  ```tsx
  useEffect(() => { if (refreshKey) mutate(); }, [refreshKey, mutate]);
  ```
- No loading spinner needed — `useJournalCalendar` uses `keepPreviousData: true` so previous month data stays visible during fetch.

**3. `journal-timeline-card.tsx`** — Single entry card:
- `"use client"` component.
- Props: `entry: JournalEntry`, `onClick: () => void`.
- Import `MOODS` from `@/components/journal/journal-mood-selector` for emoji lookup.
- Import `getPreviewText` from `@/lib/journal/utils` for content preview.
- Render a clickable Card with:
  - Top row: mood emoji (from MOODS array) + formatted date (use `new Date(entry.entry_date + "T00:00:00")` to avoid timezone shift, then `toLocaleDateString()`) + optional title.
  - Body: content preview using `getPreviewText(entry.content as Record<string, unknown>, 150)`.
  - Use `cursor-pointer` and `hover:bg-accent transition-colors duration-150` on the card.
  - Use `useTranslations("journal")` for any labels.

**4. `journal-timeline.tsx`** — Timeline feed with Load More:
- `"use client"` component.
- Props: `onEntryClick: (date: string) => void`.
- State: `allEntries: JournalEntry[]` (accumulated), `cursor: string | null`.
- Use `useJournalTimeline(cursor)` hook.
- On entries change (via `useEffect`), append new entries to `allEntries` with deduplication by `entry_date`.
- Expose `resetTimeline()` via a callback or export for parent to call on modal close (use `useImperativeHandle` with `forwardRef`, or simpler: accept an `onMutate` callback prop that the parent can call `mutate()` + reset).
  - Simpler approach: accept a `refreshKey: number` prop. When it changes (via useEffect), reset `allEntries` to `[]`, reset `cursor` to `null`, and call the hook's `mutate()`. Parent increments `refreshKey` on modal close.
- "Load More" button: shown when `hasMore` is true. On click, set `cursor` to the last entry's `entry_date`.
- Empty state: show a centered message when `allEntries` is empty and not loading. Use `useTranslations("journal")`.
- Each entry rendered via `JournalTimelineCard` with `onClick` calling `onEntryClick(entry.entry_date)`.
- Loading state: show a spinner or skeleton below entries when `isLoading`.
