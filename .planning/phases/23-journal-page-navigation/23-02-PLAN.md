  <action>
Rewrite `journal-page-content.tsx` to replace the empty-state placeholder with the full browsing experience:

**Imports:**
- Add: `Tabs, TabsList, TabsTrigger, TabsContent` from `@/components/ui/tabs`
- Add: `CalendarDays, List` from `lucide-react` (tab icons)
- Add: `JournalCalendar` from `@/components/journal/journal-calendar`
- Add: `JournalTimeline` from `@/components/journal/journal-timeline`
- Keep: `JournalEntryModal`, `PageHeader`, `Button`, `PenLine`, `BookOpen`, `getLocalDateString`, `useState`, `useTranslations`, `useCallback`

**State:**
- `modalOpen: boolean` (existing)
- `selectedDate: string` (existing, but change from useState with initializer to a settable state)
- `refreshKey: number` — incremented on modal close to trigger calendar + timeline data refresh

**Handlers:**
- `handleDayClick(date: string)` — Set `selectedDate` to the clicked date, open the modal.
- `handleEntryClick(date: string)` — Same as handleDayClick (clicking timeline entry opens modal for that date).
- `handleWriteToday()` — Set `selectedDate` to `getLocalDateString()`, open the modal.
- `handleModalClose(open: boolean)` — If `open` is false (modal closing), increment `refreshKey`. Call `setModalOpen(open)`.

**Layout:**
```
PageHeader (title + "Write Today" button)
Tabs defaultValue="calendar"
  TabsList
    TabsTrigger value="calendar" — CalendarDays icon + t("calendar")
    TabsTrigger value="timeline" — List icon + t("timeline")
  TabsContent value="calendar"
    JournalCalendar onDayClick={handleDayClick}
  TabsContent value="timeline"
    JournalTimeline onEntryClick={handleEntryClick} refreshKey={refreshKey}
JournalEntryModal open={modalOpen} onOpenChange={handleModalClose} date={selectedDate}
```

**Styling:**
- Tabs container: center the TabsList for a clean look.
- Calendar TabsContent: flex justify-center to center the calendar widget.
- Timeline TabsContent: max-w-2xl mx-auto for readable width.
- Remove the old empty-state placeholder div entirely.
- Remove the `BookOpen` import if no longer used in the page (it was only for the empty-state icon). Check — if not used, remove.

**Data refresh on modal close:**
The `refreshKey` number prop on `JournalTimeline` resets its accumulated entries and calls `mutate()` when changed. For `JournalCalendar`, since it uses `useJournalCalendar` internally, the SWR global mutate or a similar mechanism is needed. Options:
- Simplest: Pass `refreshKey` to `JournalCalendar` as well. Inside JournalCalendar, add a `useEffect` watching `refreshKey` that calls the hook's `mutate()`.
- This ensures both views refresh after modal create/edit/delete.
