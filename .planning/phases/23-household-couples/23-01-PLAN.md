Create a single migration file with ALL schema changes for Phase 23:

**1. household_invitations table:**
```sql
CREATE TABLE household_invitations (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  household_id UUID NOT NULL REFERENCES households(id) ON DELETE CASCADE,
  invited_by UUID NOT NULL REFERENCES profiles(id),
  email TEXT NOT NULL,
  token UUID NOT NULL DEFAULT gen_random_uuid() UNIQUE,
  status TEXT NOT NULL DEFAULT 'pending'
    CHECK (status IN ('pending', 'accepted', 'expired', 'revoked')),
  expires_at TIMESTAMPTZ NOT NULL DEFAULT (NOW() + INTERVAL '7 days'),
  created_at TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE(household_id, email)
);
```
Add indexes: `idx_invitations_token` on (token), `idx_invitations_email` on (email), `idx_invitations_household` on (household_id).

RLS policies:
- Enable RLS on household_invitations
- "Household owners can view invitations" -- SELECT for authenticated, USING household_id IN (SELECT household_id FROM household_members WHERE user_id = (SELECT auth.uid()) AND role = 'owner')
- "Household owners can create invitations" -- INSERT for authenticated, WITH CHECK same pattern
- "Household owners can update invitations" -- UPDATE for authenticated, USING same pattern (for revoke)

**2. Account visibility columns:**
```sql
ALTER TABLE accounts
  ADD COLUMN owner_id UUID REFERENCES profiles(id),
  ADD COLUMN visibility TEXT NOT NULL DEFAULT 'mine'
    CHECK (visibility IN ('mine', 'ours', 'hidden')),
  ADD COLUMN shared_since TIMESTAMPTZ;
```
Backfill owner_id from bank_connections.connected_by where accounts have a bank_connection_id:
```sql
UPDATE accounts SET owner_id = bc.connected_by
FROM bank_connections bc
WHERE accounts.bank_connection_id = bc.id
AND accounts.owner_id IS NULL
AND bc.connected_by IS NOT NULL;
```
For accounts without bank_connection_id (cash accounts), backfill from the first household_member (owner):
```sql
UPDATE accounts a SET owner_id = (
  SELECT hm.user_id FROM household_members hm
  WHERE hm.household_id = a.household_id AND hm.role = 'owner'
  LIMIT 1
)
WHERE a.owner_id IS NULL;
```
Add indexes: `idx_accounts_owner` on (owner_id), `idx_accounts_visibility` on (visibility).

**3. Transaction visibility columns:**
```sql
ALTER TABLE transactions
  ADD COLUMN is_hidden_from_household BOOLEAN NOT NULL DEFAULT false,
  ADD COLUMN is_shared_to_household BOOLEAN NOT NULL DEFAULT false;
```

**4. Budget/Goal ownership columns:**
```sql
ALTER TABLE budgets
  ADD COLUMN owner_id UUID REFERENCES profiles(id),
  ADD COLUMN is_shared BOOLEAN NOT NULL DEFAULT false;

ALTER TABLE savings_goals
  ADD COLUMN owner_id UUID REFERENCES profiles(id),
  ADD COLUMN is_shared BOOLEAN NOT NULL DEFAULT false;
```
Backfill budget and goal owner_id from household owner:
```sql
UPDATE budgets b SET owner_id = (
  SELECT hm.user_id FROM household_members hm
  WHERE hm.household_id = b.household_id AND hm.role = 'owner'
  LIMIT 1
)
WHERE b.owner_id IS NULL;

UPDATE savings_goals sg SET owner_id = (
  SELECT hm.user_id FROM household_members hm
  WHERE hm.household_id = sg.household_id AND hm.role = 'owner'
  LIMIT 1
)
WHERE sg.owner_id IS NULL;
```
Add indexes: `idx_budgets_owner` on (owner_id), `idx_goals_owner` on (owner_id).

```typescript
export class HouseholdsDB {
  constructor(private supabase: SupabaseClient) {}

  // Existing (moved from resolveHousehold):
  async resolveHousehold(userId: string): Promise<string>

  // New methods:
  async getMembers(householdId: string): Promise<HouseholdMemberWithProfile[]>
  // Query household_members JOIN profiles to get email, full_name, avatar_url

  async getMemberRole(householdId: string, userId: string): Promise<string | null>
  // Single-row query for role

  async getMemberCount(householdId: string): Promise<number>
  // Count query

  async createInvite(householdId: string, invitedBy: string, email: string): Promise<HouseholdInvitation>
  // Insert into household_invitations, handle 23505 (duplicate)

  async getInvitations(householdId: string): Promise<HouseholdInvitation[]>
  // Select pending invitations for household

  async getInvitationByToken(token: string): Promise<HouseholdInvitation | null>
  // Select by token, check not expired, not already accepted
  // Uses admin client (no RLS filter on invitee's email)

  async acceptInvite(token: string, userId: string, adminClient: SupabaseClient): Promise<void>
  // 1. Validate token (exists, pending, not expired)
  // 2. Check household size limit (max 5)
  // 3. Get user's current household membership
  // 4. If user is sole owner of a solo household, merge:
  //    - Update all owned accounts: SET household_id = target, keep owner_id, SET visibility = 'mine'
  //    - Update all transactions for those accounts: SET household_id = target
  //    - Update all budgets: SET household_id = target, SET is_shared = false
  //    - Update all savings_goals: SET household_id = target, SET is_shared = false
  //    - Update all recurring_bills: SET household_id = target
  //    - Update all manual_assets: SET household_id = target
  //    - Update merchant_category_rules: SET household_id = target
  //    - Move bank_connections: SET household_id = target
  //    - Handle categories: skip duplicates by name, remap transactions
  //    - Delete net_worth_snapshots for source (recalculate later)
  //    - Update household_members: SET household_id = target, role = 'member'
  //    - Delete source household
  // 5. If user has no household, just insert as member
  // 6. Mark invitation as 'accepted'

  async removeMember(householdId: string, userId: string, adminClient: SupabaseClient): Promise<void>
  // Reverse merge: create new solo household for departing member
  // 1. Create new household
  // 2. Move departing member's owned accounts + their transactions to new household
  // 3. Move departing member's budgets (owner_id match) to new household, set is_shared = false
  // 4. Move departing member's goals (owner_id match) to new household, set is_shared = false
  // 5. Move departing member's bank_connections to new household
  // 6. Update household_members: new household, role = 'owner'
  // 7. Leave shared budgets/goals in original household (transfer ownership to household owner)

  async revokeInvite(invitationId: string): Promise<void>
  // Update status to 'revoked'

  async getPendingInvitesForEmail(email: string, adminClient: SupabaseClient): Promise<HouseholdInvitation[]>
  // Used on login to check if user has pending invites
}

// Backward compatibility wrapper:
export async function resolveHousehold(supabase: SupabaseClient, userId: string): Promise<string> {
  const db = new HouseholdsDB(supabase);
  return db.resolveHousehold(userId);
}
```

IMPORTANT: The merge flow in `acceptInvite` and the reverse flow in `removeMember` MUST use `adminClient` (service role) since they update data across household boundaries that RLS would block.
