**Migration 1: `20260221000001_create_households.sql`**

Create `households` and `household_members` tables following the pattern from `20260219000001_create_projects_table.sql`:

```sql
-- households table
CREATE TABLE households (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name TEXT NOT NULL DEFAULT 'My Household',
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- household_members table
CREATE TABLE household_members (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  household_id UUID NOT NULL REFERENCES households(id) ON DELETE CASCADE,
  user_id UUID NOT NULL REFERENCES profiles(id) ON DELETE CASCADE,
  role TEXT NOT NULL DEFAULT 'owner' CHECK (role IN ('owner', 'member')),
  created_at TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE(household_id, user_id)
);
```

Add indexes: `idx_household_members_user_id` on `household_members(user_id)`, `idx_household_members_household_id` on `household_members(household_id)`.

Enable RLS on both tables. Create policies:
- `households` SELECT: `id IN (SELECT household_id FROM household_members WHERE user_id = (SELECT auth.uid()))`
- `household_members` SELECT: `user_id = (SELECT auth.uid())`
- `household_members` INSERT: `user_id = (SELECT auth.uid())` — users can only add themselves (invitations handled in Phase 23)
- `households` INSERT: `TO authenticated` with no restriction (any authenticated user can create a household)

Add `update_households_updated_at` trigger reusing existing `update_updated_at_column()` function.

**Money arithmetic: `lib/money/arithmetic.ts`**

Create using the exact implementation from RESEARCH.md Pattern 4:
- `Decimal.set({ precision: 20, rounding: Decimal.ROUND_HALF_UP })`
- `toCents(dollars: number | string): number` — multiply by 100, round
- `formatMoney(cents: number): string` — format as `$1,234.56` or `-$1,234.56` with always 2 decimals, comma grouping, minus prefix for negatives. Use `Intl.NumberFormat("en-US", ...)` for comma grouping instead of `toLocaleString` to ensure consistent behavior in Node.js
- `centsToDecimal(cents: number): string` — raw decimal string (e.g., "10.33")
- `addCents(a: number, b: number): number` — integer addition
- `subtractCents(a: number, b: number): number` — integer subtraction

Per user decision: always USD ($), always two decimals, negative = minus prefix (-$1,234.56), standard comma grouping.
