    **1. `lib/hooks/use-accounts.ts`** — SWR hook:
    ```typescript
    import useSWR from "swr";
    import { fetcher } from "@/lib/fetcher";

    export function useAccounts() {
      const { data, error, isLoading, mutate } = useSWR(
        "/api/money/accounts",
        fetcher,
        { keepPreviousData: true }
      );
      return { data, error, isLoading, mutate };
    }
    ```
    The response shape: `{ connections: ConnectionWithAccounts[], net_worth_cents: number }` where ConnectionWithAccounts includes accounts array and derived sync_status.

      - stale: `bg-money-amber-light text-money-amber-foreground` with "Stale" text
      - error: `bg-destructive/10 text-destructive` with "Error" text
    - Use `useTranslations('money')` for translated status text

      - SyncStatusBadge positioned top-right
      - Account subtype as small label (e.g., "Checking", "Credit Card")

    **4. `components/money/account-group.tsx`** — "use client":
    - Props: `connection: ConnectionWithAccounts`, `onSync`, `onDisconnect`
    - Institution header with institution_name in bold
    - Per-institution subtotal (sum of account balances)
    - Re-sync button (RefreshCw icon) — always visible per locked decision
    - Disconnect button (Unlink icon) — opens disconnect dialog
    - List of AccountCard components
    - If connection has error status, show error_message below header

    **5. `components/money/accounts-list.tsx`** — "use client":
    - Uses `useAccounts()` SWR hook
    - Top section: Net worth summary card showing formatted total
    - Below: list of AccountGroup components for each connection
    - Loading state: skeleton cards while fetching

    - Connect Bank button (PlaidLinkButton) in header area — this is the primary CTA per locked decision
    - "Other options" dropdown below Plaid button for manual entry (secondary, per locked decision)

    **6. `components/money/plaid-link-button.tsx`** — "use client":
    - Fetches link_token from `/api/money/plaid/create-link-token` on mount (or on demand)
    - Uses `usePlaidLink({ token: linkToken, onSuccess, onExit })` from `react-plaid-link`
    - `onSuccess(publicToken)`: calls `/api/money/plaid/exchange-token` with public_token, shows success toast via sonner, calls `mutate` to refresh accounts list
    - `onExit(err)`: if err, shows error toast; if cancelled, shows cancellation toast
    - Button styled as primary CTA using Calm Finance tokens
    - Loading state while fetching link_token
    - Disabled state while link is initializing

    **7. `components/money/disconnect-dialog.tsx`** — "use client":
    - AlertDialog pattern (same as ProjectDeleteDialog)
    - Props: `connectionId`, `institutionName`, `open`, `onOpenChange`
    - Two options: "Keep Transactions" and "Delete Transactions"
    - Calls `/api/money/accounts/{id}/disconnect` with `{ keep_transactions }` body
    - Shows toast on success, refreshes accounts list via mutate

    **8. `components/money/accounts-empty-state.tsx`** — "use client":
