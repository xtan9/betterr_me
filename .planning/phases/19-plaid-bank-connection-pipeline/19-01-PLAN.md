    Also create Vault wrapper functions in the same migration for accessing Vault from the Supabase JS client:

    ```sql
    -- Vault wrapper functions (callable via admin.rpc())
    CREATE OR REPLACE FUNCTION create_plaid_secret(secret_name TEXT, secret_value TEXT, secret_description TEXT DEFAULT '')
    RETURNS UUID
    LANGUAGE plpgsql
    SECURITY DEFINER
    AS $$
    DECLARE
      secret_id UUID;
    BEGIN
      SELECT vault.create_secret(secret_value, secret_name, secret_description) INTO secret_id;
      RETURN secret_id;
    END;
    $$;

    CREATE OR REPLACE FUNCTION get_plaid_secret(secret_name TEXT)
    RETURNS TEXT
    LANGUAGE plpgsql
    SECURITY DEFINER
    AS $$
    DECLARE
      result TEXT;
    BEGIN
      SELECT decrypted_secret INTO result
      FROM vault.decrypted_secrets
      WHERE name = secret_name;
      RETURN result;
    END;
    $$;

    CREATE OR REPLACE FUNCTION delete_plaid_secret(secret_name TEXT)
    RETURNS VOID
    LANGUAGE plpgsql
    SECURITY DEFINER
    AS $$
    BEGIN
      DELETE FROM vault.secrets WHERE name = secret_name;
    END;
    $$;
    ```

    These SECURITY DEFINER functions run with the definer's privileges, so they can access `vault.create_secret` and `vault.decrypted_secrets` even when called via the PostgREST API (admin client).

    **1. `lib/plaid/types.ts`** — TypeScript types for Plaid integration:

    **2. `lib/plaid/client.ts`** — Plaid API client factory:
    ```typescript
    import { Configuration, PlaidApi, PlaidEnvironments } from 'plaid';

    export function createPlaidClient(): PlaidApi {
      const configuration = new Configuration({
        basePath: PlaidEnvironments[process.env.PLAID_ENV as 'sandbox' | 'production']
          || PlaidEnvironments.sandbox,
        baseOptions: {
          headers: {
            'PLAID-CLIENT-ID': process.env.PLAID_CLIENT_ID!,
            'PLAID-SECRET': process.env.PLAID_SECRET!,
            'Plaid-Version': '2020-09-14',
          },
        },
      });
      return new PlaidApi(configuration);
    }
    ```
    NOT a singleton — follows project convention (create new instances, not module-level singletons).

    **3. `lib/plaid/token-exchange.ts`** — Token exchange + Vault storage:
    - `exchangeAndStore(publicToken, householdId, userId, supabaseAdmin)`:
      1. Call `plaidClient.itemPublicTokenExchange({ public_token: publicToken })`
      2. Get `access_token` and `item_id`
      3. Call `plaidClient.itemGet({ access_token })` for institution info
      4. Call `plaidClient.accountsGet({ access_token })` for accounts list
      5. Store access token in Vault via `supabaseAdmin.rpc('create_plaid_secret', { secret_name, secret_value, secret_description })`
      6. Insert `bank_connections` row with vault_secret_name, plaid_item_id, institution_name, status='connected', connected_by=userId
      7. Insert `accounts` rows for each Plaid account with plaid_account_id, name, balance (converted via `toCents`), subtype, mask
      8. Return the created bank_connection and accounts
    - `getAccessToken(bankConnectionId, supabaseAdmin)`:
      1. Query `bank_connections` for `vault_secret_name`
      2. Call `supabaseAdmin.rpc('get_plaid_secret', { secret_name: vaultSecretName })`
      3. Return decrypted access token
    - `removeAccessToken(bankConnectionId, supabaseAdmin)`:
      1. Query `bank_connections` for `vault_secret_name`
      2. Call `supabaseAdmin.rpc('delete_plaid_secret', { secret_name: vaultSecretName })`

    Use `toCents` from `@/lib/money/arithmetic` for Plaid balance conversion. Remember to **negate** Plaid balances for liability accounts (credit cards have positive current balance in Plaid, which represents debt = negative in our system).

    **4. `lib/plaid/sync.ts`** — Cursor-based transaction sync:
    - `syncTransactions(accessToken, cursor, bankConnectionId, householdId, supabaseAdmin)`:
      1. Loop `plaidClient.transactionsSync({ access_token, cursor })` while `has_more`
      2. Collect added, modified, removed arrays
      3. For added: convert to our format (negate Plaid amount via `toCents(-amount)`, map plaid_account_id to our account_id, set source='plaid', extract PFCv2 categories)
      4. For modified: upsert matching `plaid_transaction_id`
      5. For removed: delete by `plaid_transaction_id`
      6. Batch upsert via admin client (bypasses RLS for webhook/cron context)
      7. Update `bank_connections` with new cursor and `last_synced_at`
      8. Return count of processed transactions and new cursor

    **5. `lib/db/bank-connections.ts`** — `BankConnectionsDB` class:
    - Constructor takes SupabaseClient (same pattern as HabitsDB, TasksDB)
    - Methods: `getByHousehold(householdId)`, `getById(id)`, `create(data)`, `updateStatus(id, status, errorCode?, errorMessage?)`, `updateSyncCursor(id, cursor, lastSyncedAt)`, `disconnect(id)` (set status='disconnected')
