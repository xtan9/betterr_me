  <action>
    Follow existing API route patterns from the project (try/catch, log.error, NextResponse.json, fresh supabase client per request).

    **1. `app/api/money/plaid/create-link-token/route.ts`** — POST:
    - Auth check: `createClient()` -> `supabase.auth.getUser()` -> 401 if no user
    - Create Plaid client: `createPlaidClient()`
    - Call `plaid.linkTokenCreate()` with:
      - `user: { client_user_id: user.id }`
      - `client_name: 'BetterR.Me'`
      - `products: [Products.Transactions]`
      - `country_codes: [CountryCode.Us]`
      - `language: 'en'`
      - `webhook: \`${process.env.NEXT_PUBLIC_APP_URL}/api/money/plaid/webhook\``
      - `redirect_uri: process.env.PLAID_REDIRECT_URI` (if set; omit for sandbox)
    - Return `{ link_token: response.data.link_token }`

    **2. `app/api/money/plaid/exchange-token/route.ts`** — POST:
    - Auth check
    - Validate body with `exchangeTokenSchema`
    - Resolve household: `resolveHousehold(supabase, user.id)`
    - Call `exchangeAndStore(body.public_token, householdId, user.id, adminClient)` from `lib/plaid/token-exchange.ts`
    - Return `{ success: true, bank_connection, accounts }`
    - Error handling: if exchange fails, return 500 with error message

    **3. `app/api/money/plaid/webhook/route.ts`** — POST:
    - **NO auth check** — webhooks come from Plaid, not authenticated users
    - Read raw body as text: `const body = await request.text()`
    - Read `Plaid-Verification` header
    - Call `verifyPlaidWebhook(body, plaidVerification, plaidClient)` — if false, return 401
    - Parse body as JSON after verification
    - Handle webhook types:
      - `TRANSACTIONS` + `SYNC_UPDATES_AVAILABLE`: look up bank_connection by plaid_item_id, call `syncTransactions()` via admin client
      - `TRANSACTIONS` + `INITIAL_UPDATE`: same as above
      - `TRANSACTIONS` + `HISTORICAL_UPDATE`: same as above
      - `ITEM` + `ERROR`: update bank_connection status to 'error' with error_code/error_message
      - `ITEM` + `PENDING_EXPIRATION`: update status to 'error' with appropriate message
    - **Always return 200 quickly** — acknowledge receipt. Sync runs after acknowledgment (within same execution, not deferred). Plaid retries on non-200.
    - Use `log.info` for webhook receipt, `log.error` for processing failures

    **4. `app/api/money/accounts/route.ts`** — GET:
    - Auth check
    - Resolve household
    - Query bank_connections with their accounts (use BankConnectionsDB + MoneyAccountsDB)
    - For each bank_connection, derive sync status:
      - `'synced'` if status='connected' AND last_synced_at > (now - 24 hours)
      - `'stale'` if status='connected' AND last_synced_at <= (now - 24 hours) OR last_synced_at is null
      - `'error'` if status='error'
    - Group accounts by bank_connection (institution)
    - Calculate net worth: sum of all account balances (checking/savings positive, credit card negative)
    - Return `{ connections: [...], net_worth_cents: number }`

    **5. `app/api/money/accounts/[id]/sync/route.ts`** — POST:
    - Auth check
    - Get bank_connection by ID (verify it belongs to user's household via RLS)
    - Verify status is 'connected' (can't sync disconnected/errored without re-auth)
    - Get access token from Vault via `getAccessToken()`
    - Call `syncTransactions()` with current cursor
    - Return `{ success: true, transactions_synced: count }`

    **6. `app/api/money/accounts/[id]/disconnect/route.ts`** — POST:
    - Auth check
    - Validate body: `{ keep_transactions: boolean }`
    - Get bank_connection by ID (verify via RLS)
    - Get access token from Vault
    - Call `plaidClient.itemRemove({ access_token })` to revoke Plaid access
    - Remove access token from Vault via `removeAccessToken()`
    - If `!keep_transactions`: delete all transactions where account_id IN (accounts for this bank_connection) AND source='plaid'
    - Update bank_connection status to 'disconnected'
    - Return `{ success: true }`

    **1. `app/api/money/transactions/route.ts`**:

    **GET** — List transactions:
    - Auth check, resolve household
    - Query via TransactionsDB.getByHousehold with optional filters: account_id, date_from, date_to, category, search
    - Return `{ transactions: [...] }`

    **POST** — Create manual transaction:
    - Auth check, resolve household
    - Validate body with `manualTransactionSchema`
    - Verify account_id belongs to user's household
    - If account_id is "cash" or missing, auto-create or find Cash account (lazy creation like resolveHousehold pattern):
      - Check for existing account with name='Cash' and bank_connection_id=NULL in this household
      - If not found, create it
    - Insert transaction with source='manual'
    - Return `{ transaction }` with 201 status

    **2. `app/api/cron/sync-transactions/route.ts`** — GET:
    - Verify `Authorization: Bearer ${CRON_SECRET}` header — 401 if missing/wrong
    - Use admin client (bypasses RLS — cron has no user session)
    - Query all bank_connections where status='connected'
    - For each connection:
      - Get access token from Vault
      - Call `syncTransactions()` with stored cursor
      - On success: update last_synced_at and cursor
      - On error: update status to 'error', log error, continue to next connection
    - Return `{ synced: N, errors: M }`
