**2. Create `lib/db/categories-db.ts`** — CategoriesDB class:

Methods:
- `getAll(householdId: string)`: Returns all system categories (household_id IS NULL) plus household custom categories. Ordered by is_system DESC, name ASC.
- `getVisible(householdId: string)`: Same as getAll but excludes hidden categories for this household. Uses LEFT JOIN or NOT IN subquery on hidden_categories.
- `create(data: CategoryInsert)`: Insert a custom category (must have household_id set).
- `update(id: string, data: Partial<Pick<Category, 'name' | 'icon' | 'color' | 'display_name'>>)`: Update category fields.
- `delete(id: string)`: Delete a custom category. Must verify `is_system = false` before deleting.
- `hide(householdId: string, categoryId: string)`: Insert into hidden_categories.
- `unhide(householdId: string, categoryId: string)`: Delete from hidden_categories.
- `getHidden(householdId: string)`: Get list of hidden category IDs for a household.

**3. Create `lib/db/merchant-rules.ts`** — MerchantRulesDB class:

Methods:
- `getByHousehold(householdId: string)`: List all rules for household, joined with categories for display. Order by merchant_name ASC.
- `findByMerchant(householdId: string, merchantName: string)`: Look up rule by merchant_name_lower (case-insensitive exact match).
- `create(data: MerchantCategoryRuleInsert)`: Insert new rule. Auto-compute `merchant_name_lower = data.merchant_name.toLowerCase()`.
- `delete(id: string)`: Delete a rule.
- `findBulk(householdId: string, merchantNames: string[])`: Look up rules for multiple merchants at once (for sync pipeline). Returns Map<string, MerchantCategoryRule>.

**4. Create `lib/db/transaction-splits.ts`** — TransactionSplitsDB class:

Methods:
- `getByTransaction(transactionId: string)`: Get all splits for a transaction, joined with categories.
- `create(splits: TransactionSplitInsert[])`: Batch insert splits for a transaction. Caller must validate amounts sum to original.
- `deleteByTransaction(transactionId: string)`: Delete all splits for a transaction (used before re-splitting).
