All API routes follow the existing project pattern: createClient() → auth check → resolveHousehold() → business logic → NextResponse.json. Accept `?date=YYYY-MM-DD` query param for timezone-safe date handling (project convention).

**`app/api/money/dashboard/route.ts` — GET:**
1. Auth + resolveHousehold
2. Parse `date` and `view` (mine|household) query params
3. Make bulk queries (NOT sequential per-item):
   - AccountsDB.getByHousehold(householdId, view) → get all accounts, compute totalBalanceCents
   - BillsDB.getByHousehold(householdId, view) → get active bills, filter upcoming 30 days using getBillDatesInRange helper
   - TransactionsDB.getByHousehold(householdId, { dateFrom: 30 days ago, dateTo: today }) → for spending rate calculation
   - Query `confirmed_income_patterns` WHERE household_id = householdId AND needs_reconfirmation = false
4. Compute server-side:
   - `dailySpendingRateCents` using computeDailySpendingRate from projections.ts
   - `availableMoneyConts` using computeAvailableMoney
   - `upcomingBillsTotalCents` — sum of absolute bill amounts in next 30 days
   - `endOfMonthBalanceCents` using computeEndOfMonthBalance
   - `dailyBalances` using projectDailyBalances (for calendar danger zones) — project from today to end of next month
   - `incomeStatus`: { detected: DetectedIncome[] | null, confirmed: ConfirmedIncomePattern[] | null, needsConfirmation: boolean }
5. Run income detection if no confirmed patterns exist: query last 6 months of positive transactions, call detectIncomePatterns, set needsConfirmation = true if patterns found
6. Return JSON:
```json
{
  "available_cents": number,
  "upcoming_bills_total_cents": number,
  "end_of_month_balance_cents": number,
  "daily_balances": DailyBalance[],
  "upcoming_bills": { merchant_name, amount_cents, due_date }[],
  "income_status": { detected, confirmed, needs_confirmation },
  "has_confirmed_income": boolean,
  "confidence_label": "estimated" | "based on confirmed income"
}
```

**`app/api/money/dashboard/summary/route.ts` — GET:**
This is the lightweight endpoint for the money summary card on the main habit/task dashboard. Must be fast and independent.

1. Auth + resolveHousehold
2. Parse `date` query param
3. Query: count of money_accounts for this household (for has_accounts check)
4. If no accounts: return `{ has_accounts: false }` immediately (fast path)
5. If accounts exist:
   - Query transactions WHERE transaction_date = today → sum abs(amount_cents) WHERE amount_cents < 0 → spent_today_cents
   - Query transactions WHERE transaction_date >= start_of_week → sum abs(amount_cents) WHERE amount_cents < 0 → spent_this_week_cents
   - Query current month budget total_limit_cents → budget_total_cents (null if no budget)
6. Return: `{ has_accounts: true, spent_today_cents, spent_this_week_cents, budget_total_cents }`

**`app/api/money/income/route.ts` — GET + POST:**

GET:
1. Auth + resolveHousehold
2. Query confirmed_income_patterns for this household
3. Also run detectIncomePatterns on last 6 months of positive transactions
4. Return: `{ confirmed: ConfirmedIncomePattern[], detected: DetectedIncome[] }`

POST (confirm income pattern):
1. Auth + resolveHousehold
2. Validate body with incomeConfirmationSchema from validations/money.ts
3. Upsert into confirmed_income_patterns (household_id, merchant_name unique)
4. Set needs_reconfirmation = false, confirmed_at = now()
5. Return 200 with confirmed pattern

POST with `action: "dismiss"`:
1. Set needs_reconfirmation = false and DELETE the pattern if user rejects detection
2. Or better: just don't store if user dismisses. The absence of confirmation means "not confirmed."

**`app/api/money/insights/route.ts` — GET + POST:**

GET:
1. Auth + resolveHousehold
2. Parse `page` query param (optional): "dashboard" | "budgets" | "bills" | "goals"
3. Fetch data for insight computation:
   - Current month spending by category (from TransactionsDB aggregation)
   - Previous 3 months spending by category (for historical average)
   - Recurring bills with previous_amount_cents (for subscription alerts)
   - Active goals with projections (for goal progress insights)
   - Dismissed insight IDs from dismissed_insights table for this household
4. Compute insights using functions from lib/money/insights.ts:
   - computeSpendingAnomalies(currentMonth, historicalAvg, currentMonthStr)
   - computeSubscriptionAlerts(bills)
   - computeGoalInsights(goals)
   - computeInsights({ spendingAnomalies, subscriptionAlerts, goalInsights, dismissedIds })
5. If `page` provided, filter insights to only that page
6. Return: `{ insights: Insight[] }`

POST (dismiss insight):
1. Auth + resolveHousehold
2. Validate body with insightDismissSchema
3. INSERT INTO dismissed_insights (household_id, insight_id) ON CONFLICT DO NOTHING
4. Return 200

**`lib/hooks/use-dashboard-money.ts`:**
```typescript
"use client";
import useSWR from "swr";
import { fetcher } from "@/lib/fetcher"; // or inline fetcher like other hooks
import type { ViewMode } from "@/lib/db/types";
import { getLocalDateString } from "@/lib/utils";

// Response type matching dashboard API
interface DashboardMoneyData {
  available_cents: number;
  upcoming_bills_total_cents: number;
  end_of_month_balance_cents: number;
  daily_balances: DailyBalance[];
  upcoming_bills: { merchant_name: string; amount_cents: number; due_date: string }[];
  income_status: { detected: DetectedIncome[] | null; confirmed: ConfirmedIncomePattern[] | null; needs_confirmation: boolean };
  has_confirmed_income: boolean;
  confidence_label: "estimated" | "based on confirmed income";
}

export function useDashboardMoney(view: ViewMode = "mine") {
  const date = getLocalDateString();
  const { data, error, mutate } = useSWR<DashboardMoneyData>(
    `/api/money/dashboard?date=${date}&view=${view}`,
    fetcher,
    { keepPreviousData: true }
  );
  return { dashboard: data ?? null, isLoading: !data && !error, error, mutate };
}
```

**`lib/hooks/use-insights.ts`:**
```typescript
"use client";
import useSWR from "swr";
import type { Insight, InsightPage } from "@/lib/db/types";

export function useInsights(page?: InsightPage) {
  const key = page ? `/api/money/insights?page=${page}` : `/api/money/insights`;
  const { data, error, mutate } = useSWR<{ insights: Insight[] }>(key, fetcher);
  const dismiss = async (insightId: string) => {
    await fetch("/api/money/insights", {
      method: "POST",
      headers: { "Content-Type": "application/json" },
      body: JSON.stringify({ insight_id: insightId }),
    });
    mutate(); // Revalidate to remove dismissed insight
  };
  return { insights: data?.insights ?? [], isLoading: !data && !error, dismiss };
}
```

```typescript
"use client";
import useSWR from "swr";
import { getLocalDateString } from "@/lib/utils";

interface MoneySummary {
  spent_today_cents: number;
  spent_this_week_cents: number;
  budget_total_cents: number | null;
  has_accounts: boolean;
}

export function useMoneySummary() {
  const date = getLocalDateString();
  const { data, error } = useSWR<MoneySummary>(
    `/api/money/dashboard/summary?date=${date}`,
    fetcher,
    { shouldRetryOnError: false, revalidateOnFocus: false, refreshInterval: 300_000 }
  );
  return {
    summary: data ?? null,
    isLoading: !data && !error,
    hasAccounts: data?.has_accounts ?? false,
  };
}
```
