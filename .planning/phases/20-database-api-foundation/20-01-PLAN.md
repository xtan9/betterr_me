**Migration 1: journal_entries table** (`supabase/migrations/20260222100001_create_journal_entries.sql`)

Create the journal_entries table following the exact pattern from `20260219000001_create_projects_table.sql`:

```sql
CREATE TABLE journal_entries (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID NOT NULL REFERENCES profiles(id) ON DELETE CASCADE,
  entry_date DATE NOT NULL,
  title TEXT NOT NULL,
  content JSONB NOT NULL DEFAULT '{"type":"doc","content":[]}',
  mood INTEGER NOT NULL DEFAULT 3 CHECK (mood BETWEEN 1 AND 5),
  word_count INTEGER NOT NULL DEFAULT 0,
  tags TEXT[] NOT NULL DEFAULT '{}',
  prompt_key TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE (user_id, entry_date)
);
```

**Migration 2: journal_entry_links table** (`supabase/migrations/20260222100002_create_journal_entry_links.sql`)

Create the polymorphic links table:

```sql
CREATE TABLE journal_entry_links (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  entry_id UUID NOT NULL REFERENCES journal_entries(id) ON DELETE CASCADE,
  link_type TEXT NOT NULL CHECK (link_type IN ('habit', 'task', 'project')),
  link_id UUID NOT NULL,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE (entry_id, link_type, link_id)
);
```

Add indexes: `(entry_id)` for fetching links by entry, `(link_type, link_id)` for reverse lookup.

Enable RLS with 3 policies (SELECT, INSERT, DELETE -- no UPDATE needed for links) using EXISTS subquery on journal_entries to check user ownership. No user_id column on this table (per research recommendation).

**JournalEntriesDB class** (`lib/db/journal-entries.ts`)

Follow the `ProjectsDB` pattern in `lib/db/projects.ts`. Constructor takes `SupabaseClient`. Implement these methods:

1. `upsertEntry(entry: JournalEntryInsert): Promise<JournalEntry>` — Uses `.upsert(entry, { onConflict: 'user_id,entry_date' }).select().single()`. CRITICAL: entry data MUST include user_id and entry_date for onConflict to work.

2. `getEntryByDate(userId: string, date: string): Promise<JournalEntry | null>` — Uses `.maybeSingle()` (NOT `.single()`) since entry may not exist for a date. Filter by user_id and entry_date.

3. `getEntry(entryId: string, userId: string): Promise<JournalEntry | null>` — Get by ID. Uses `.single()`. Handle PGRST116 error code returning null (same pattern as ProjectsDB.getProject).

4. `updateEntry(entryId: string, userId: string, updates: JournalEntryUpdate): Promise<JournalEntry>` — Uses `.update(updates).eq('id', entryId).eq('user_id', userId).select().single()`.

5. `deleteEntry(entryId: string, userId: string): Promise<void>` — Uses `.delete().eq('id', entryId).eq('user_id', userId)`. Throw on error.

6. `getCalendarMonth(userId: string, year: number, month: number): Promise<JournalCalendarDay[]>` — Select ONLY `entry_date, mood, title` (never full content). Filter by user_id, entry_date >= start of month, entry_date <= end of month. Use zero-padded month string. Order ascending.

7. `getTimeline(userId: string, limit = 10, cursor?: string): Promise<JournalEntry[]>` — Select all fields. Order by entry_date DESC. If cursor provided, filter `.lt('entry_date', cursor)`. Limit results.

Import { log } from '@/lib/logger' and use log.error for error context where appropriate. All methods throw on Supabase errors (let API routes catch).

**JournalEntryLinksDB class** (`lib/db/journal-entry-links.ts`)

Constructor takes `SupabaseClient`. Methods:

1. `getLinksForEntry(entryId: string): Promise<JournalEntryLink[]>` — Select all links for an entry, ordered by created_at.

2. `addLink(link: JournalEntryLinkInsert): Promise<JournalEntryLink>` — Insert, select, single. Throw on error (including unique constraint violation for duplicate links).

3. `removeLink(linkId: string, entryId: string): Promise<void>` — Delete by id AND entry_id. Throw on error.
