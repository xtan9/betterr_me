## Previously-Critical Breaks — Now Resolved

### Break 1 (FIXED): `title` missing from autosave POST payload

- **Original issue:** `journalEntryFormSchema` required `title: z.string().trim().min(1)` but autosave never sent title
- **Fix (Phase 26):** Changed to `title: z.string().trim().max(200).default("")` — missing title defaults to empty string
- **Verification:** Zod schema confirmed at `lib/validations/journal.ts:7-11`; test "should succeed with default empty title when title omitted" passes with 201

### Break 2 (FIXED): `mood: null` rejected by Zod schema

- **Original issue:** `mood: z.number().default(3)` rejected null (`.default()` only replaces undefined)
- **Fix (Phase 26):** Changed to `mood: z.number().int().min(1).max(5).nullable().default(null)` + DB migration drops NOT NULL
- **Verification:** Schema confirmed at `lib/validations/journal.ts:13`; tests pass for mood: null on both POST and PATCH
- **Note:** Migration `20260223200001_make_mood_nullable.sql` must be applied to production Supabase
