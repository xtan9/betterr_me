    1. **`playBeep(frequency = 440, durationMs = 200): void`**
       - Lazy-create AudioContext on first call (NOT at module scope — Pitfall 2)
       - Store AudioContext in module-level variable for reuse
       - Resume if suspended: `if (audioContext.state === "suspended") audioContext.resume()`
       - Create OscillatorNode: frequency configurable, type 'sine', gain 0.3 (not too loud)
       - Start immediately, stop after durationMs
       - Wrap entire function in try/catch, silent fail if audio unavailable

    2. **`useRestTimer()` hook** — returns `{ remaining, isActive, start, adjust, skip }`:
       - `remaining: number` — seconds remaining (0 when inactive)
       - `isActive: boolean` — whether timer is counting down
       - `start(durationSeconds: number)` — sets `endTime = Date.now() + durationSeconds * 1000`
       - `adjust(deltaSeconds: number)` — modifies endTime by delta (clamped so remaining >= 0). For REST-03 +15s/-15s buttons.
       - `skip()` — sets endTime to null, stops timer
       - Internal: uses `setInterval(tick, 200)` where tick computes `remaining = Math.max(0, Math.ceil((endTime - Date.now()) / 1000))`. 200ms interval for smooth display.
       - When remaining reaches 0: call `playBeep()`, then after a configurable auto-dismiss delay (e.g., 3 seconds), set endTime to null.
       - `visibilitychange` listener: on document visible, force a tick recalculation. This ensures REST-05 (correct time after tab switch). Clean up listener on unmount.
       - Uses `useState` for endTime and remaining, `useCallback` for start/adjust/skip, `useEffect` for the interval.

    Add `"use client"` directive since this file contains React hooks.

    IMPORTANT anti-patterns to avoid:
    - Do NOT use `remaining--` tick-counting (Pitfall from RESEARCH.md)
    - Do NOT create AudioContext at module scope
    - Do NOT use `setInterval` tick count as source of truth — always recompute from `Date.now()`
  </action>
