  <action>
    Create `lib/hooks/use-active-workout.ts` (per RESEARCH.md Pattern 3):

    ```typescript
    "use client";
    import useSWR from "swr";
    import type { WorkoutWithExercises } from "@/lib/db/types";
    import { saveWorkoutToStorage, clearWorkoutStorage } from "@/lib/fitness/workout-session";
    ```

    1. **`useActiveWorkout()` hook** — returns `{ workout, error, isLoading, mutate, actions }`:
       - SWR key: `"/api/workouts/active"` with fetcher that extracts `data.workout`
       - SWR config: `revalidateOnFocus: false` (client is sole writer during active session), `dedupingInterval: 60000`
       - `actions` object with mutation methods that call the API + optimistic SWR update + localStorage dual-write:
         - `startWorkout(title?)`: POST /api/workouts → mutate cache → save to localStorage
         - `addExercise(exerciseId, restTimerSeconds?)`: POST /api/workouts/[id]/exercises → optimistic mutate → save
         - `removeExercise(workoutExerciseId)`: DELETE → optimistic mutate → save
         - `addSet(workoutExerciseId)`: POST /api/workouts/[id]/exercises/[weId]/sets → optimistic mutate → save
         - `updateSet(workoutExerciseId, setId, updates)`: PATCH → optimistic mutate (revalidate: false per Pitfall 4) → save
         - `deleteSet(workoutExerciseId, setId)`: DELETE → optimistic mutate → save
         - `completeSet(workoutExerciseId, setId)`: shortcut for updateSet with `{ is_completed: true }` — also returns true to signal rest timer should start
         - `updateExerciseNotes(workoutExerciseId, notes)`: PATCH exercise → optimistic mutate → save
         - `updateWorkout(updates)`: PATCH /api/workouts/[id] → optimistic mutate → save
         - `finishWorkout()`: PATCH status:'completed' → clear localStorage → mutate null
         - `discardWorkout()`: PATCH status:'discarded' → clear localStorage → mutate null

       **Optimistic update pattern:** Use `mutate(key, async () => { /* api call, return updated workout */ }, { optimisticData: updatedWorkout, revalidate: false })`. For set updates, use `revalidate: false` to prevent Pitfall 4 race conditions.

       **Dual-write:** After each successful API call, call `saveWorkoutToStorage(updatedWorkout)`. On finish/discard, call `clearWorkoutStorage()`.

    2. **`useWeightUnit()` hook** — small helper that reads weight_unit from `/api/profile` SWR cache:
       ```typescript
       export function useWeightUnit(): WeightUnit {
         const { data } = useSWR<{ weight_unit?: string }>("/api/profile", fetcher);
         return (data?.weight_unit as WeightUnit) ?? "kg";
       }
       ```
       This extracts weight_unit from the profile preferences. Fallback to "kg" if not set. The profile API already exists and is cached by the sidebar user footer.

    Use `useCallback` for all action methods to maintain stable references.
  </action>
