  <name>Task 1: Create fitness database tables migration with all 6 tables</name>
  <files>supabase/migrations/20260224000001_create_fitness_tables.sql</files>
  <action>
Create a single migration file containing ALL 6 fitness tables in dependency order:

1. **exercises** — with columns: id (UUID PK), user_id (UUID FK profiles, nullable for presets), name (TEXT NOT NULL), muscle_group_primary (TEXT NOT NULL), muscle_groups_secondary (TEXT[] DEFAULT '{}'), equipment (TEXT NOT NULL DEFAULT 'barbell'), exercise_type (TEXT NOT NULL DEFAULT 'weight_reps'), is_custom (BOOLEAN NOT NULL DEFAULT false), created_at, updated_at.
   - CHECK constraint on exercise_type: 8 values (weight_reps, bodyweight_reps, weighted_bodyweight, assisted_bodyweight, duration, duration_weight, distance_duration, weight_distance)
   - CHECK constraint on equipment: 9 values (barbell, dumbbell, machine, bodyweight, kettlebell, cable, band, other, none)
   - CHECK constraint on muscle_group_primary: 16 values (chest, back, shoulders, biceps, triceps, forearms, core, quadriceps, hamstrings, glutes, calves, traps, lats, full_body, cardio, other)
   - Indexes: idx_exercises_user (user_id WHERE is_custom = true), idx_exercises_muscle (muscle_group_primary), idx_exercises_preset (id WHERE user_id IS NULL)
   - RLS: SELECT allows `user_id IS NULL OR auth.uid() = user_id`, INSERT/UPDATE/DELETE require `auth.uid() = user_id AND is_custom = true`

2. **routines** — id, user_id (NOT NULL FK profiles CASCADE), name, notes, last_performed_at, created_at, updated_at. Standard user-owned RLS (auth.uid() = user_id for all ops). Index on user_id. Updated_at trigger.

3. **workouts** — id, user_id (NOT NULL FK profiles CASCADE), title (DEFAULT 'Workout'), started_at (TIMESTAMPTZ NOT NULL DEFAULT NOW()), completed_at, duration_seconds (INTEGER), status (TEXT NOT NULL DEFAULT 'in_progress' CHECK in_progress/completed), notes, routine_id (UUID FK routines SET NULL), created_at, updated_at. UNIQUE partial index on (user_id) WHERE status = 'in_progress'. Index on (user_id, started_at DESC). Index on routine_id WHERE NOT NULL. Standard user-owned RLS. Updated_at trigger.

4. **workout_exercises** — id, workout_id (NOT NULL FK workouts CASCADE), exercise_id (NOT NULL FK exercises RESTRICT), sort_order (DOUBLE PRECISION DEFAULT 0), notes, rest_timer_seconds (INTEGER DEFAULT 90), created_at. Index on (workout_id, sort_order). RLS via EXISTS subquery joining to workouts WHERE user_id = auth.uid().

5. **workout_sets** — id, workout_exercise_id (NOT NULL FK workout_exercises CASCADE), set_number (INTEGER NOT NULL), set_type (TEXT NOT NULL DEFAULT 'normal' CHECK warmup/normal/drop/failure), weight_kg (NUMERIC(7,2) nullable), reps (INTEGER nullable), duration_seconds (INTEGER nullable), distance_meters (NUMERIC(10,2) nullable), is_completed (BOOLEAN DEFAULT false), rpe (SMALLINT CHECK 1-10 or NULL), created_at, updated_at. Index on (workout_exercise_id, set_number). RLS via EXISTS subquery joining workout_exercises -> workouts. Updated_at trigger.

6. **routine_exercises** — id, routine_id (NOT NULL FK routines CASCADE), exercise_id (NOT NULL FK exercises RESTRICT), sort_order (DOUBLE PRECISION DEFAULT 0), target_sets (INTEGER DEFAULT 3), target_reps (INTEGER nullable), target_weight_kg (NUMERIC(7,2) nullable), target_duration_seconds (INTEGER nullable), rest_timer_seconds (INTEGER DEFAULT 90), notes, created_at. Index on (routine_id, sort_order). RLS via EXISTS subquery joining routines.

IMPORTANT: Create tables in this exact order (routines before workouts, because workouts references routines via routine_id FK). Use `update_updated_at_column()` function which already exists in the database from initial schema migration.

Reference the exact SQL from `.planning/research/ARCHITECTURE.md` for column definitions, constraints, and RLS policies.
