    **Step 1: Create `app/workouts/[id]/page.tsx`.**
    Server component page route:
    - Extract `id` from params (use `await params` pattern for Next.js 16)
    - Authenticate via `createClient()` + `supabase.auth.getUser()`
    - Fetch workout using `WorkoutsDB(supabase).getWorkoutWithExercises(id)` -- this method already exists from Phase 19
    - If workout not found or not owned by user, return `notFound()` (from next/navigation)
    - If workout status is not "completed", redirect to `/workouts/active` (they should use the active workout page instead)

    - Use `getTranslations("workouts")` for server-side translations

    **Step 2: Create `components/fitness/workout-history/workout-detail-view.tsx`.**
    Client component ("use client") that renders a read-only view of a completed workout:
    - Props: `workout: WorkoutWithExercises` (existing type from lib/db/types), `weightUnit: WeightUnit`
    - Summary section at top: total duration, total exercises, total sets, total volume in a horizontal stat strip
    - For each exercise in the workout:
      - Exercise name as a heading
      - Exercise notes (if any) in text-muted-foreground
      - A table/grid showing all sets: set number, weight (in user's unit), reps, duration, set type badge (warmup/normal/drop/failure), completion status
      - **PR badges on qualifying sets** (see Step 2b below)
      - Use similar layout to WorkoutExerciseCard from the active workout logger but in read-only mode (no inputs, no buttons)
      - Set type badges use same color coding as active workout: warmup=yellow, normal=default, drop=purple, failure=red
    - Workout notes section at the bottom (if notes exist)
    - Use `formatWeight` and `displayWeight` from `lib/fitness/units` for weight display
    - Use `useTranslations("workouts")` for labels

    **Step 2b: Add PR badges to sets in the detail view.**
    For each exercise in the workout, determine which sets achieved personal records:
    - Import `useExerciseRecords` from `lib/hooks/use-exercise-history` and `computePersonalRecords` from `lib/fitness/personal-records`
    - For each exercise, use `useExerciseRecords(exercise.exercise_id)` to fetch the current all-time PRs
    - Compare each completed set against the personal records to identify PR sets. A set is a PR if its weight, volume (weight * reps), reps, or duration equals the corresponding PR value AND the set's workout date matches or is after the PR was first achieved. Use a simpler heuristic: compare set values against the PR values -- if a set's weight equals best_weight_kg, mark it as a weight PR; if reps equals best_reps, mark as reps PR; etc.
    - Render a small "PR" badge (inline, amber/gold themed: `bg-amber-100 text-amber-800 dark:bg-amber-900/30 dark:text-amber-400 text-xs font-medium px-1.5 py-0.5 rounded`) next to qualifying sets in the set table
    - Only show PR badges for the relevant PR types per exercise type (using `EXERCISE_FIELD_MAP` from `lib/fitness/exercise-fields`)
    - If records are still loading, do not show badges (graceful degradation)
