**Update `app/workouts/page.tsx`:**

The workouts landing page is currently a server component. To show routine cards (which need SWR data), extract the page content into a client component wrapper. Follow the same pattern used for the exercise library page (server component page wrapper + client component content).

Create a client component section (either inline or as a separate file like `components/fitness/workouts-page-content.tsx`) that:

1. Uses `useRoutines()` to fetch routines.
2. Renders a "My Routines" section after the WorkoutResumeBanner:
   - Section heading with a link to `/workouts/routines` ("Manage Routines" or "View All" link).
   - If routines exist, display up to 3 routine cards in a horizontal scrolling row or a grid (grid-cols-1 sm:grid-cols-2 lg:grid-cols-3). Each `RoutineCard` has a "Start" button that:
     - Calls POST `/api/routines/[id]/start`.
     - On success (201): navigates to `/workouts/active` via `router.push`.
     - On 409: shows toast "You already have an active workout".
     - On other error: shows toast with error message.
   - If no routines, show a subtle prompt: "Create your first routine to quickly start workouts" with a link to `/workouts/routines`.
3. Keeps the existing exercise library Card below the routines section.

The page.tsx server component should:
- Still use `getTranslations` for the PageHeader.
- Render PageHeader, StartWorkoutButton, and WorkoutResumeBanner as before.
- Render the new client content component for routines.
- Keep the exercise library Card.

**Update `components/fitness/workout-logger/workout-finish-dialog.tsx`:**

Add a "Save as Routine" feature to the finish dialog:

1. Add state: `const [showSaveRoutine, setShowSaveRoutine] = useState(false)` and `const [routineName, setRoutineName] = useState("")` and `const [saving, setSaving] = useState(false)`.
2. After the summary stats section and before the confirm/cancel buttons, add a "Save as Routine" expandable section:
   - A button/link text "Save as Routine" (LayoutTemplate icon) that toggles `showSaveRoutine`.
   - When expanded: show a text input for routine name (pre-fill with workout title if available), and a "Save" button.
   - On save: POST to `/api/workouts/[workoutId]/save-as-routine` with `{ name: routineName }`. Show toast on success ("Routine saved!") or error. Collapse the section after success.
3. The save-as-routine action should work independently of the finish action — user can save as routine and then still click "Finish Workout" to complete the workout.
4. Only show "Save as Routine" when the workout has at least one exercise.
5. Use `useTranslations("routines")` for the save-as-routine strings (or use the workouts namespace with a `saveAsRoutine` sub-key).
