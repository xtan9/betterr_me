Create `lib/db/routines.ts` with a `RoutinesDB` class following the WorkoutsDB/CategoriesDB pattern:

**Constructor:** Takes `SupabaseClient` as parameter.

**Methods:**

1. `getUserRoutines(userId: string): Promise<RoutineWithExercises[]>` — Select from `routines` with nested `routine_exercises(*, exercise:exercises(*))`. Filter by `user_id`. Order routines by `updated_at DESC`. Order routine_exercises by `sort_order ASC` using Supabase's `referencedTable` option. Map the Supabase response to reshape the nested `exercise` field.

2. `getRoutine(routineId: string): Promise<RoutineWithExercises | null>` — Same nested select but filtered by `id`. Return null if not found (catch PGRST116 error code). Used by detail/update/delete routes.

3. `createRoutine(userId: string, data: { name: string; notes?: string | null }): Promise<Routine>` — Insert into `routines` table with `user_id` and fields from data. Return the created routine.

4. `updateRoutine(routineId: string, updates: RoutineUpdate): Promise<Routine>` — Update routine by id. Return updated routine. Throw if not found.

5. `deleteRoutine(routineId: string): Promise<void>` — Delete routine by id. CASCADE will handle routine_exercises cleanup.

6. `addExerciseToRoutine(routineId: string, data: { exercise_id: string; target_sets?: number; target_reps?: number | null; target_weight_kg?: number | null; target_duration_seconds?: number | null; rest_timer_seconds?: number; notes?: string | null }): Promise<RoutineExercise>` — Compute next `sort_order` using the same max+65536 pattern from `WorkoutExercisesDB.addExercise`. Insert into `routine_exercises`. Default `target_sets` to 3, `rest_timer_seconds` to 90.

7. `updateRoutineExercise(routineExerciseId: string, updates: object): Promise<RoutineExercise>` — Update routine_exercise fields (target_sets, target_reps, target_weight_kg, target_duration_seconds, rest_timer_seconds, notes, sort_order).

8. `removeRoutineExercise(routineExerciseId: string): Promise<void>` — Delete routine_exercise by id.

All methods should use `log.error` on failure and throw the Supabase error. Import `log` from `@/lib/logger`.

Import types from `./types`: `Routine`, `RoutineUpdate`, `RoutineExercise`, `RoutineWithExercises`.

**Update `lib/db/index.ts`:** Add `export * from "./routines";` after the workout-exercises export.

Create 4 API route files following the exact same auth/validation/error-handling pattern from existing workout API routes.

**`app/api/routines/route.ts`** — GET + POST:
- GET: Auth check -> `new RoutinesDB(supabase).getUserRoutines(user.id)` -> return `{ routines }`.
- POST: Auth check -> `validateRequestBody(body, routineCreateSchema)` -> `routinesDB.createRoutine(user.id, validation.data)` -> return `{ routine }` with status 201.
- Error handling: try/catch with `log.error` and 500 response.

**`app/api/routines/[id]/route.ts`** — GET + PATCH + DELETE:
- All three: Auth check first.
- GET: `routinesDB.getRoutine(id)` -> return `{ routine }` or 404.
- PATCH: `validateRequestBody(body, routineUpdateSchema)` -> `routinesDB.updateRoutine(id, validation.data)` -> return `{ routine }`.
- DELETE: `routinesDB.deleteRoutine(id)` -> return `{ success: true }`.
- Use `const { id } = await params;` for Next.js 16 dynamic route params.
- Error handling: try/catch with log.error. On PGRST116 (not found), return 404.

**`app/api/routines/[id]/exercises/route.ts`** — GET + POST:
- GET: Auth check -> Get routine exercises (can call `routinesDB.getRoutine(routineId)` and return just exercises, or do a direct query). Return `{ exercises }`.
- POST: Auth check -> `validateRequestBody(body, routineExerciseAddSchema)` -> `routinesDB.addExerciseToRoutine(routineId, validation.data)` -> return `{ exercise }` with status 201.
- Use `const { id } = await params;` for the routine ID.

**`app/api/routines/[id]/exercises/[reId]/route.ts`** — PATCH + DELETE:
- PATCH: Auth check -> `validateRequestBody(body, routineExerciseUpdateSchema)` -> `routinesDB.updateRoutineExercise(reId, validation.data)` -> return `{ exercise }`.
- DELETE: Auth check -> `routinesDB.removeRoutineExercise(reId)` -> return `{ success: true }`.
- Use `const { id, reId } = await params;` for both IDs.
