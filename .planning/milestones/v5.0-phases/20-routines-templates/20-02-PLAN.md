**`app/api/routines/[id]/start/route.ts`** — POST handler implementing copy-on-start:

1. Auth check (get user from supabase.auth.getUser, return 401 if not authenticated).
2. Extract routine ID from `await params`.
3. Instantiate `RoutinesDB` and `WorkoutsDB` and `WorkoutExercisesDB`.
4. Fetch routine with exercises via `routinesDB.getRoutine(routineId)`. Return 404 if not found.
5. Create workout via direct supabase insert (not via WorkoutsDB.startWorkout because we need to set routine_id and title from routine name):
   ```
   Insert into workouts: { user_id, title: routine.name, status: 'in_progress', started_at: new Date().toISOString(), routine_id: routineId }
   ```
   Catch error code `23505` and return 409 with `{ error: "You already have an active workout" }` (same pattern as POST /api/workouts).
6. For each routine exercise (sorted by sort_order):
   - Insert into `workout_exercises`: `{ workout_id, exercise_id, sort_order, notes, rest_timer_seconds }` — copy from routine exercise.
   - Create pre-filled sets: For `i = 1` to `target_sets`, insert into `workout_sets`: `{ workout_exercise_id, set_number: i, set_type: 'normal', weight_kg: target_weight_kg, reps: target_reps, duration_seconds: target_duration_seconds, is_completed: false }`. Use `EXERCISE_FIELD_MAP` logic to only copy applicable fields: if the exercise type doesn't show weight, set `weight_kg` to null; if it doesn't show reps, set `reps` to null; if it doesn't show duration, set `duration_seconds` to null. Import `EXERCISE_FIELD_MAP` from `@/lib/fitness/exercise-fields` and look up the exercise's `exercise_type`.
7. Update routine's `last_performed_at` to `new Date().toISOString()`.
8. Return the created workout with `{ workout }` and status 201.

**`app/api/workouts/[id]/save-as-routine/route.ts`** — POST handler:

1. Auth check.
2. Extract workout ID from `await params`.
3. Validate body with `saveAsRoutineSchema` (requires `name` field).
4. Instantiate `WorkoutsDB` and `RoutinesDB`.
5. Fetch the workout with exercises via a nested Supabase select:
   ```
   supabase.from('workouts').select('*, workout_exercises(*, exercise:exercises(*), sets:workout_sets(*))').eq('id', workoutId).single()
   ```
   Return 404 if not found. Return 400 if workout status is 'in_progress' (only completed/discarded workouts can be saved as routines).
6. Create routine: `supabase.from('routines').insert({ user_id, name: validation.data.name }).select().single()`.
7. For each workout exercise (sorted by sort_order):
   - Find the best completed set values: filter sets where `is_completed = true`, find max `weight_kg`, use first completed set's `reps` as target, use first completed set's `duration_seconds` as target.
   - Compute `sort_order` using sequential 65536 gaps.
   - Insert into `routine_exercises`: `{ routine_id, exercise_id, sort_order, target_sets: completedSets.length || sets.length, target_reps, target_weight_kg, target_duration_seconds, rest_timer_seconds, notes }`.
8. Return `{ routine }` with status 201.

Both routes follow standard error handling: try/catch -> `log.error` -> 500 response.

**`lib/hooks/use-routines.ts`** — SWR hook following useExercises pattern:
```typescript
import useSWR from "swr";
import { fetcher } from "@/lib/fetcher";
import type { RoutineWithExercises } from "@/lib/db/types";

export function useRoutines() {
  const { data, error, isLoading, mutate } = useSWR<{ routines: RoutineWithExercises[] }>(
    "/api/routines",
    fetcher,
    { revalidateOnFocus: false, dedupingInterval: 30000 }
  );
  return { routines: data?.routines ?? [], error, isLoading, mutate };
}
```
