Follow the exact patterns from `app/api/projects/route.ts` and `app/api/projects/[id]/route.ts`.

**POST /api/journal** (`app/api/journal/route.ts`)

1. Auth check: `const supabase = await createClient(); const { data: { user } } = await supabase.auth.getUser(); if (!user) return 401`
2. Parse and validate body with `validateRequestBody(body, journalEntryFormSchema)`
3. `await ensureProfile(supabase, user)`
4. `const journalDB = new JournalEntriesDB(supabase)`
5. Call `journalDB.upsertEntry()` with user_id from auth, entry_date and all validated fields from body. Set defaults: mood ?? 3, word_count ?? 0, tags ?? [], prompt_key ?? null
6. Return `{ entry }` with status 201 (always 201, per research recommendation)
7. Catch: `log.error('POST /api/journal error', error)`, return 500

**GET /api/journal** (`app/api/journal/route.ts`)

1. Auth check
2. Extract `date` from `request.nextUrl.searchParams`
3. If no date param, return 400 with `{ error: 'date query parameter is required' }`
4. Validate date format with regex `/^\d{4}-\d{2}-\d{2}$/`
5. Call `journalDB.getEntryByDate(user.id, date)`
6. Return `{ entry }` (entry can be null -- no 404, null means "no entry for this date")
7. Catch: `log.error('GET /api/journal error', error)`, return 500

**GET /api/journal/[id]** (`app/api/journal/[id]/route.ts`)

1. Auth check
2. Extract id from `await params` (Next.js 15+ async params pattern, same as projects/[id])
3. Call `journalDB.getEntry(id, user.id)`
4. If null, return 404 with `{ error: 'Journal entry not found' }`
5. Return `{ entry }` with status 200
6. Catch: log and return 500

**PATCH /api/journal/[id]** (`app/api/journal/[id]/route.ts`)

1. Auth check
2. Extract id from `await params`
3. Validate body with `validateRequestBody(body, journalEntryUpdateSchema)`
4. Call `journalDB.updateEntry(id, user.id, validation.data)`
5. Return `{ entry }` with status 200
6. Catch: log and return 500. Handle PGRST116 as 404.

**DELETE /api/journal/[id]** (`app/api/journal/[id]/route.ts`)

1. Auth check
2. Extract id from `await params`
3. Call `journalDB.deleteEntry(id, user.id)`
4. Return `{ success: true }` with status 200
5. Catch: log and return 500

**GET /api/journal/calendar** (`app/api/journal/calendar/route.ts`)

1. Auth check
2. Extract `year` and `month` from search params
3. Validate: both required, year is a number, month is 1-12
4. Call `journalDB.getCalendarMonth(user.id, Number(year), Number(month))`
5. Return `{ entries }` with status 200
6. Catch: log and return 500

All routes use `import { log } from '@/lib/logger'` (NOT console.error). All routes use `import { createClient } from '@/lib/supabase/server'`. All routes instantiate DB class per-request (no singletons).
