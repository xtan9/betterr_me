  ```
    TaskList,
  ```

  ```
  const onUpdateRef = useRef(onUpdate);
  useEffect(() => { onUpdateRef.current = onUpdate; }, [onUpdate]);
  ```

**Step 3: Create `lib/hooks/use-journal-autosave.ts`.**
Debounced autosave hook following Pattern 3 from RESEARCH.md:
- `"use client"` directive
- Import `useRef`, `useState`, `useCallback`, `useEffect` from `react`
- Export `SaveStatus` type alias (same as journal-save-status)
- Hook signature: `useJournalAutosave(entryId: string | null, entryDate: string, delay = 2000)`
- State: `saveStatus` (SaveStatus), initialized to `"idle"`
- Refs: `timeoutRef` (setTimeout ID), `pendingRef` (queued data), `entryIdRef` (tracks current entry ID for stale closure prevention)
- `entryIdRef` updates when entryId prop changes (useEffect)
- `save` callback:
  - Sets status to `"saving"`
  - Determines URL and method: if entryIdRef.current exists → `PATCH /api/journal/${entryIdRef.current}`, else → `POST /api/journal`
  - Body: for POST includes `entry_date: entryDate`, for PATCH just the data
  - On success: set status `"saved"`, return `result.entry` (importantly: update entryIdRef.current with returned entry.id for subsequent saves)
  - On error: set status `"error"`, return null
- `scheduleSave` callback:
  - Store data in pendingRef
  - Clear existing timeout
  - Set new timeout (delay ms) that calls save with pendingRef.current
- `flushNow` callback:
  - If pendingRef.current exists, clear timeout and immediately call save
- `beforeunload` handler (useEffect):
  - On beforeunload: if pendingRef.current, use `navigator.sendBeacon` with `new Blob([JSON.stringify(pendingRef.current)], { type: 'application/json' })` (Pitfall 7 fix)
  - Cleanup: remove listener, clear timeout
- Return: `{ saveStatus, scheduleSave, flushNow, savedEntryId: entryIdRef.current }`

IMPORTANT: The hook must handle the "first save creates the entry, subsequent saves update it" transition. After the first POST succeeds and returns an entry with an id, store that id in entryIdRef so the next save uses PATCH.
