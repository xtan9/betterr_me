**`components/money/csv-import-dialog.tsx`** — Multi-step wizard dialog for CSV import:

"use client" component using `useTranslations("money.csvImport")`.

**State management:** `step` (1-4: upload, map, preview, importing/done), `file`, `parsedHeaders`, `parsedRows`, `columnMapping`, `flipSign`, `selectedAccountId`, `importResult`, `isImporting`, `open`.

**Step 1 — Upload & Account Selection:**
- File input (`accept=".csv,.tsv,.txt"`) with drag-drop zone or simple input
- Account selector dropdown using `useAccounts()` hook from `lib/hooks/use-accounts.ts` — list accounts + "Cash" option
- "Flip sign" checkbox (default unchecked) for CSVs with opposite sign convention
- On file select: parse with PapaParse (`Papa.parse(file, { header: true, skipEmptyLines: "greedy", beforeFirstChunk: (chunk) => chunk.replace(/^\uFEFF/, "") })`)
- On parse complete: store headers, rows (limit preview to first 100 via `parsedRows.slice(0, 100)`), auto-map columns via `autoMapColumns(headers)`, advance to step 2
- Show error if file has 0 data rows or >5000 rows (display `MAX_IMPORT_ROWS` from csv-import.ts)

**Step 2 — Column Mapping:**
- For each target field in `TARGET_FIELDS`, show a Select dropdown (from shadcn/ui) with the CSV headers as options + "Skip" option (null)
- Pre-filled from `autoMapColumns()` result. `transaction_date` and `amount` are required (show red asterisk). `description` is required.
- Show the first 3 CSV rows as a preview table with current mapping applied
- "Back" and "Next" buttons. "Next" validates that date, amount, description are all mapped.

**Step 3 — Preview:**
- Show table of mapped data (first 20 rows): Date, Amount (formatted), Description, Merchant, Category columns
- If `flipSign` is checked, negate displayed amounts
- Show total row count: "Importing {N} transactions to {accountName}"
- "Back" and "Import" buttons

**Step 4 — Importing / Result:**
- Show `Loader2` spinner with "Importing..." text while POST is in progress
- POST to `/api/money/transactions/import` with body: `{ account_id: selectedAccountId, rows: mappedRows, skip_duplicates: true }`
  - `mappedRows` maps each parsedRow using columnMapping: `{ transaction_date: row[mapping.transaction_date], amount: parseFloat(row[mapping.amount]) * (flipSign ? -1 : 1), description: row[mapping.description], merchant_name: mapping.merchant_name ? row[mapping.merchant_name] : null, category: mapping.category ? row[mapping.category] : null }`
  - Filter out rows where date is invalid or amount is NaN before sending
- On success: show `imported` count, `duplicates_skipped` count. `toast.success(t("importSuccess", { count: result.imported }))`. Close dialog. Trigger SWR revalidation via `mutate` on transaction keys.
- On error: `toast.error(t("importError"))`. Stay on step 4 with retry option.
