**`lib/money/projections.ts`:**

```typescript
import { addDays, differenceInDays, format, endOfMonth } from "date-fns";
```

Functions:
1. `projectDailyBalances(input: { currentBalanceCents: number; upcomingBills: { amount_cents: number; due_date: string }[]; dailySpendingRateCents: number; confirmedIncome: { amount_cents: number; next_date: string; frequency: string }[] | null; startDate: string; endDate: string; }): DailyBalance[]`
   - Walk forward day by day from startDate to endDate
   - Each day: subtract dailySpendingRateCents from running balance
   - On bill due dates: subtract bill amount_cents (these are stored as negative, so ADD them — same sign convention as project: negative = outflow)
   - Wait — bills amount_cents are negative in our DB. So adding them subtracts. Document clearly.
   - On income dates: add income amount_cents (positive)
   - For recurring income, project next occurrences based on frequency (WEEKLY +7d, BIWEEKLY +14d, SEMI_MONTHLY: 1st and 15th, MONTHLY: same date next month)
   - Return array of DailyBalance objects

2. `computeAvailableMoney(currentBalanceCents: number, billsDueBeforePayday: { amount_cents: number }[], nextPayDate: string | null, today: string): number`
   - If nextPayDate: available = currentBalance + sum(bills before payday) — bills are negative, so sum adds negative = subtracts
   - If no nextPayDate: use end-of-month as fallback
   - Returns available cents (can be negative)

3. `computeEndOfMonthBalance(currentBalanceCents: number, dailySpendingRateCents: number, daysRemaining: number, upcomingBillsCents: number, expectedIncomeCents: number): number`
   - Simple: current + expectedIncome + upcomingBills (neg) - (dailyRate * daysRemaining)
   - Sign convention: upcomingBillsCents is negative (sum of bill amounts), expectedIncomeCents is positive

4. `computeDailySpendingRate(transactions: { amount_cents: number; transaction_date: string }[], startDate: string, endDate: string): number`
   - Filter transactions with amount_cents < 0 (outflows only)
   - Sum absolute values, divide by number of days in range
   - Returns positive cents/day

5. `getDangerZoneStatus(projectedBalanceCents: number, dailySpendingRateCents: number): "safe" | "tight" | "danger"`
   - danger: projected <= 0
   - tight: projected > 0 but < 2 * dailySpendingRateCents
   - safe: otherwise

**`lib/money/income-detection.ts`:**

```typescript
import { differenceInDays, format, addDays, addMonths } from "date-fns";
```

Function:
1. `detectIncomePatterns(transactions: { merchant_name: string | null; amount_cents: number; transaction_date: string }[]): DetectedIncome[]`
   - Filter: amount_cents > 0 only (income/deposits)
   - Filter out transactions without merchant_name (can't track pattern without merchant)
   - Group by normalized merchant_name (trim, lowercase)
   - For each group with >= 3 occurrences:
     a. Sort by transaction_date ascending
     b. Compute intervals between consecutive dates (differenceInDays)
     c. Compute median interval
     d. Classify frequency: 5-9d = WEEKLY, 12-16d = BIWEEKLY or SEMI_MONTHLY, 26-35d = MONTHLY
     e. Compute confidence = 1 - (stddev of intervals / median interval), clamp [0, 1]
     f. Compute median amount_cents across all occurrences
     g. Predict next_date based on last_occurrence + frequency interval
   - Return patterns with confidence >= 0.7
   - Sort by amount_cents descending (largest income first)

Helper: `predictNextIncomeDate(lastDate: string, frequency: string): string` — adds appropriate interval

**`lib/money/insights.ts`:**

```typescript
import type { Insight, RecurringBill, BudgetWithCategories, GoalWithProjection } from "@/lib/db/types";
```

Functions:
1. `computeSpendingAnomalies(currentMonthSpending: { category_name: string; amount_cents: number }[], historicalAverage: { category_name: string; avg_cents: number }[], currentMonth: string): Insight[]`
   - For each category: compute percent_change = ((current - avg) / avg) * 100
   - If abs(percent_change) >= 15%: create insight
   - type: "spending_anomaly", page: "budgets", severity: percent_change > 0 ? "attention" : "positive"
   - data: { category: category_name, percent_change: Math.round(percent_change), period: "3-month average" }
   - id: `spending_anomaly:${category_name.toLowerCase()}:${currentMonth}`

2. `computeSubscriptionAlerts(bills: RecurringBill[]): Insight[]`
   - Filter bills where previous_amount_cents is not null AND previous_amount_cents !== amount_cents
   - For each: compute price change percentage
   - type: "subscription_increase", page: "bills", severity: "attention"
   - data: { merchant: merchant_name, old_amount_cents, new_amount_cents, percent_change }
   - id: `subscription_increase:${merchant_name.toLowerCase()}:${format(new Date(), "yyyy-MM")}`

3. `computeGoalInsights(goals: GoalWithProjection[]): Insight[]`
   - For each active goal:
     - If projected_date exists and deadline exists and projected_date > deadline: "behind schedule" insight
     - Compute progress inline: `(goal.current_cents / goal.target_cents) * 100`. If progress >= 90: "almost there" insight. NOTE: GoalWithProjection does NOT have a `progress_percent` field — always compute inline from `current_cents / target_cents`.
   - type: "goal_progress", page: "goals", severity varies
   - id: `goal_progress:${goal.id}:${format(new Date(), "yyyy-MM")}`

4. `computeInsights(params: { spendingAnomalies: Insight[]; subscriptionAlerts: Insight[]; goalInsights: Insight[]; dismissedIds: Set<string>; }): Insight[]`
   - Merge all insights, filter out dismissed IDs, sort by severity (attention first), limit per page (max 5)

5. `generateInsightId(type: string, entity: string, period: string): string`
   - Returns `${type}:${entity.toLowerCase().replace(/\s+/g, '_')}:${period}`
