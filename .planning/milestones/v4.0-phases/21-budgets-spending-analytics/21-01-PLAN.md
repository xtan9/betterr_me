**Migration SQL** (`supabase/migrations/20260223000001_create_budgets.sql`):

Create two tables following the existing money table patterns (BIGINT cents, household_id FK, IN-subquery RLS):

```sql
-- budgets: one row per household per month (envelope container)
CREATE TABLE budgets (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  household_id UUID NOT NULL REFERENCES households(id) ON DELETE CASCADE,
  month DATE NOT NULL,  -- always first day of month, e.g. '2026-02-01'
  total_cents BIGINT NOT NULL,
  rollover_enabled BOOLEAN NOT NULL DEFAULT false,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE(household_id, month)
);

-- budget_categories: per-category allocation within a budget envelope
CREATE TABLE budget_categories (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  budget_id UUID NOT NULL REFERENCES budgets(id) ON DELETE CASCADE,
  category_id UUID NOT NULL REFERENCES categories(id) ON DELETE CASCADE,
  allocated_cents BIGINT NOT NULL,
  rollover_cents BIGINT NOT NULL DEFAULT 0,  -- can be negative (overspend debt carries forward)
  created_at TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE(budget_id, category_id)
);
```

Add RLS policies using the IN-subquery pattern (same as all money tables):
- `budgets`: SELECT/INSERT/UPDATE/DELETE where `household_id IN (SELECT household_id FROM household_members WHERE user_id = (SELECT auth.uid()))`
- `budget_categories`: SELECT/INSERT/UPDATE/DELETE where `budget_id IN (SELECT id FROM budgets WHERE household_id IN (SELECT household_id FROM household_members WHERE user_id = (SELECT auth.uid())))`

Add `updated_at` trigger on `budgets` table (same pattern as existing money tables).

Add index on `budgets(household_id, month)` for fast monthly lookups.

**BudgetsDB class** (`lib/db/budgets.ts`):

Create following the existing DB class pattern (constructor takes SupabaseClient):

Methods:
1. `getByMonth(householdId: string, month: string): Promise<BudgetWithCategories | null>` — Fetch budget for a specific month, joined with budget_categories. Also compute spending per category by querying transactions table (sum `amount_cents` where `amount_cents > 0` and `transaction_date` within month range, grouped by `category_id`). Handle split transactions: if a transaction has entries in `transaction_splits`, use split amounts per category instead of the parent transaction amount. Join with categories table to get `name`, `icon`, `color`.

2. `create(budget: { household_id: string; month: string; total_cents: number; rollover_enabled: boolean }, categories: { category_id: string; allocated_cents: number }[]): Promise<Budget>` — Insert budget row, then insert all budget_categories rows. Return the created budget.

3. `update(budgetId: string, updates: { total_cents?: number; rollover_enabled?: boolean }, categories?: { category_id: string; allocated_cents: number }[]): Promise<Budget>` — Update budget fields. If categories provided, delete existing budget_categories and re-insert (atomic replace, same pattern as transaction splits).

4. `delete(budgetId: string): Promise<void>` — Delete budget (CASCADE deletes budget_categories).

5. `getSpendingByCategory(householdId: string, dateFrom: string, dateTo: string): Promise<{ category_id: string; total_cents: number }[]>` — Fetch all transactions in date range where `amount_cents > 0` (outflows/spending). For transactions with splits, use split amounts per category. For transactions without splits, use the transaction's `category_id` and full `amount_cents`. Aggregate in JavaScript (Supabase JS client doesn't support GROUP BY). Filter out transactions where `category_id IS NULL`.

6. `getSpendingTrends(householdId: string, months: number): Promise<{ month: string; category_id: string; total_cents: number }[]>` — Fetch spending aggregated by month for the last N months. Used for the bar chart trend view. Calls `getSpendingByCategory` in a loop for each month.

7. `computeRollover(budgetId: string, householdId: string): Promise<{ category_id: string; rollover_cents: number }[]>` — Compute rollover amounts for each category: `allocated_cents + rollover_cents - spent_cents`. Negative results mean overspend debt. Used when user confirms rollover.

8. `confirmRollover(fromBudgetId: string, toBudgetId: string, rollovers: { category_id: string; rollover_cents: number }[]): Promise<void>` — Update `rollover_cents` on each `budget_categories` row in the target budget with the computed amounts.

Export `BudgetsDB` from `lib/db/index.ts`.

Note: The spending aggregation needs to handle the sign convention. In this project, positive `amount_cents` = outflow (spending), negative = inflow (income). Filter on `amount_cents > 0` for spending. Verify this in the implementation by checking existing transaction data patterns from Phase 19/20.

```typescript
import { z } from "zod";
import { moneyAmountSchema } from "./money";

// Category allocation within a budget envelope
export const budgetCategoryInputSchema = z.object({
  category_id: z.string().uuid(),
  amount: moneyAmountSchema.refine((v) => v >= 0, "Amount must be non-negative"),
});

// Create a new budget for a month
export const budgetCreateSchema = z
  .object({
    month: z.string().regex(/^\d{4}-\d{2}-01$/, "Must be first day of month (YYYY-MM-01)"),
    total: moneyAmountSchema.refine((v) => v > 0, "Total must be positive"),
    rollover_enabled: z.boolean().optional().default(false),
    categories: z.array(budgetCategoryInputSchema).min(1, "At least one category required"),
  })
  .refine(
    (data) => {
      const sum = data.categories.reduce((acc, c) => acc + c.amount, 0);
      return sum <= data.total;
    },
    { message: "Category allocations cannot exceed total budget", path: ["categories"] }
  );

// Update an existing budget
export const budgetUpdateSchema = z
  .object({
    total: moneyAmountSchema.refine((v) => v > 0, "Total must be positive").optional(),
    rollover_enabled: z.boolean().optional(),
    categories: z.array(budgetCategoryInputSchema).min(1).optional(),
  })
  .refine((data) => Object.keys(data).length > 0, {
    message: "At least one field required",
  })
  .refine(
    (data) => {
      // If both total and categories provided, validate envelope constraint
      if (data.total && data.categories) {
        const sum = data.categories.reduce((acc, c) => acc + c.amount, 0);
        return sum <= data.total;
      }
      return true;
    },
    { message: "Category allocations cannot exceed total budget", path: ["categories"] }
  );

// Confirm rollover from previous month to current
export const rolloverConfirmSchema = z.object({
  from_budget_id: z.string().uuid(),
});

// Query params for spending analytics
export const spendingQuerySchema = z.object({
  month: z.string().regex(/^\d{4}-\d{2}$/, "Must be YYYY-MM format"),
});

export const trendQuerySchema = z.object({
  months: z.coerce.number().int().min(1).max(24).optional().default(12),
});

// Export inferred types
export type BudgetCreateInput = z.infer<typeof budgetCreateSchema>;
export type BudgetUpdateInput = z.infer<typeof budgetUpdateSchema>;
```

Key design decisions:
- Allow sum of allocations to be LESS than total (flexible buffer per Claude's discretion — reduces anxiety, Calm Finance philosophy)
- Envelope constraint: allocations cannot EXCEED total
- Month format is `YYYY-MM-01` for creation (first day) but `YYYY-MM` for queries (simpler)
- `rollover_enabled` defaults to false
