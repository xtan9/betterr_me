**Budget list/create route** (`app/api/money/budgets/route.ts`):

Follow the existing money API pattern: `createClient -> getUser -> 401 if null -> resolveHousehold -> business logic`.

- `GET /api/money/budgets?month=YYYY-MM` â€” Returns budget for the specified month with category spending data. Query param `month` is required (YYYY-MM format). Calls `budgetsDB.getByMonth(householdId, month + '-01')`. Returns 200 with budget data or 200 with `{ budget: null }` if no budget exists for that month.

- `POST /api/money/budgets` â€” Create a new budget. Validate body with `budgetCreateSchema`. Convert dollar amounts to cents via `toCents()` before passing to `budgetsDB.create()`. Return 201 with created budget. Return 409 if a budget already exists for that household+month (UNIQUE constraint violation).

**Budget detail/update/delete route** (`app/api/money/budgets/[id]/route.ts`):

- `GET /api/money/budgets/[id]` â€” Get budget detail with category spending. Calls `budgetsDB.getByMonth` by looking up the budget's month first, or add a `getById` method to BudgetsDB. Return 404 if not found.

- `PUT /api/money/budgets/[id]` â€” Update budget. Validate with `budgetUpdateSchema`. Convert amounts to cents. If categories provided, re-validate envelope constraint against the (possibly new) total. Call `budgetsDB.update()`. Return 200.

- `DELETE /api/money/budgets/[id]` â€” Delete budget. Call `budgetsDB.delete()`. Return 204.

**Rollover confirmation route** (`app/api/money/budgets/[id]/rollover/route.ts`):

- `POST /api/money/budgets/[id]/rollover` â€” Confirm rollover from the budget identified by `[id]` (the previous month's budget) to the current/next month's budget. Validate with `rolloverConfirmSchema`. Compute rollover via `budgetsDB.computeRollover()`. Find or validate the target budget exists (the next month). Call `budgetsDB.confirmRollover()` to write rollover_cents to target budget categories. Return 200 with the computed rollover amounts.

**Budget hook** (`lib/hooks/use-budgets.ts`):

Follow the existing SWR hook pattern from `use-transactions.ts` and `use-accounts.ts`:

```typescript
"use client";
import useSWR from "swr";
import { fetcher } from "@/lib/fetcher";
import type { BudgetWithCategories } from "@/lib/db/types";

export function useBudget(month: string) {
  const { data, error, isLoading, mutate } = useSWR<{ budget: BudgetWithCategories | null }>(
    month ? `/api/money/budgets?month=${month}` : null,
    fetcher,
    { keepPreviousData: true }
  );

  return {
    budget: data?.budget ?? null,
    isLoading,
    error,
    mutate,
  };
}
```

`keepPreviousData: true` is critical â€” prevents skeleton flash when navigating between months (per project convention for date-keyed SWR).

**Spending analytics hook** (`lib/hooks/use-spending-analytics.ts`):

```typescript
"use client";
import useSWR from "swr";
import { fetcher } from "@/lib/fetcher";

interface CategorySpending {
  category_id: string;
  category_name: string;
  category_icon: string | null;
  category_color: string | null;
  total_cents: number;
}

interface MonthlyTrend {
  month: string;
  total_cents: number;
  categories: { category_id: string; total_cents: number }[];
}

export function useSpendingAnalytics(month: string) {
  const { data, error, isLoading } = useSWR<{ spending: CategorySpending[] }>(
    month ? `/api/money/analytics/spending?month=${month}` : null,
    fetcher,
    { keepPreviousData: true }
  );

  return {
    spending: data?.spending ?? [],
    isLoading,
    error,
  };
}

export function useSpendingTrends(months: number = 12) {
  const { data, error, isLoading } = useSWR<{ trends: MonthlyTrend[] }>(
    `/api/money/analytics/spending?type=trends&months=${months}`,
    fetcher,
    { keepPreviousData: true }
  );

  return {
    trends: data?.trends ?? [],
    isLoading,
    error,
  };
}
```

Both hooks use `keepPreviousData: true` to maintain Calm Finance UX (no jarring loading states when data changes).
