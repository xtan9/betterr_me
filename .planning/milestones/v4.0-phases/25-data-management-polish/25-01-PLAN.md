- `transactionsToCsv(transactions: Transaction[], accountNameMap?: Record<string, string>): string` — Generates CSV string with UTF-8 BOM prefix (`\uFEFF`) for Excel compatibility. Headers: Date, Description, Merchant, Amount, Category, Account, Source, Notes. Uses `centsToDecimal()` from `@/lib/money/arithmetic` for amount formatting. If `accountNameMap` provided, resolve `account_id` to human-readable name; otherwise use raw account_id. Import `Transaction` type from `@/lib/db/types`.

Create `lib/validations/data-management.ts` with:
- `exportTransactionsSchema` — Zod schema: `{ date_from: z.string().regex(/^\d{4}-\d{2}-\d{2}$/).optional(), date_to: z.string().regex(/^\d{4}-\d{2}-\d{2}$/).optional() }`
- `deleteMoneyDataSchema` — Zod schema: `{ confirmation: z.literal("DELETE") }`

Create `tests/lib/money/csv-export.test.ts`:

- Test `transactionsToCsv`: empty array returns just headers, single transaction produces correct row, amounts use centsToDecimal (1033 -> "10.33"), account names resolved from map, handles null merchant_name/notes gracefully, output starts with UTF-8 BOM character

  <action>
Create `app/api/money/export/route.ts` — GET endpoint:
- Auth: `const supabase = await createClient(); const { data: { user } } = await supabase.auth.getUser();` — 401 if no user.
- Parse `date_from` and `date_to` from `request.nextUrl.searchParams`. Validate with `exportTransactionsSchema`.
- Resolve household: `const householdsDB = new HouseholdsDB(supabase); const householdId = await householdsDB.resolveHousehold(user.id);`
- Fetch accounts for name map: `const accountsDB = new MoneyAccountsDB(supabase); const accounts = await accountsDB.getByHousehold(householdId);` Build `accountNameMap: Record<string, string>` from `accounts.map(a => [a.id, a.name || a.official_name || a.account_id])`.
- Fetch transactions: `const transactionsDB = new TransactionsDB(supabase); const { transactions } = await transactionsDB.getByHouseholdFiltered(householdId, user.id, "mine", { dateFrom, dateTo, limit: 10000, offset: 0 });`
- Generate CSV: `const csv = transactionsToCsv(transactions, accountNameMap);`
- Return `new Response(csv, { headers: { "Content-Type": "text/csv; charset=utf-8", "Content-Disposition": "attachment; filename=\"betterrme-transactions-${dateFrom || 'all'}-to-${dateTo || 'all'}.csv\"" } })`
- Wrap in try/catch: `log.error("GET /api/money/export error", error); return NextResponse.json({ error: "Failed to export" }, { status: 500 });`

Create `app/api/money/delete-data/route.ts` — POST endpoint:
- Auth: same pattern as above — 401 if no user.
- Parse body with `deleteMoneyDataSchema.safeParse(body)` — 400 if invalid.
- Resolve household via `HouseholdsDB(supabase).resolveHousehold(user.id)`.
- Get member count via `HouseholdsDB(supabase).getMemberCount(householdId)`.
- Create `adminClient = createAdminClient()` (from `lib/supabase/admin`).
- Create helper `async function revokeAllPlaidTokens(hId: string, admin: SupabaseClient)`:
  - Query `bank_connections` for household where status !== 'disconnected' using adminClient.
  - For each connection: try `getAccessToken(conn.id, admin)` then Plaid `itemRemove`, catch and log warning. Then try `removeAccessToken(conn.id, admin)`, catch and log warning. Same graceful pattern as existing disconnect route.
  - Import `createPlaidClient` from `@/lib/plaid/client` for `itemRemove`.
- If multi-member (count > 1):
  - `await householdsDB.removeMember(householdId, user.id, adminClient)` — this leaves household, creates new solo household
  - Resolve new household: `const newHouseholdId = await new HouseholdsDB(adminClient).resolveHousehold(user.id);`
  - `await revokeAllPlaidTokens(newHouseholdId, adminClient);`
  - Delete new household: `await adminClient.from("households").delete().eq("id", newHouseholdId);`
  - Remove membership: `await adminClient.from("household_members").delete().eq("user_id", user.id);`
- If solo (count <= 1):
  - `await revokeAllPlaidTokens(householdId, adminClient);`
  - `await adminClient.from("household_members").delete().eq("household_id", householdId);`
  - `await adminClient.from("households").delete().eq("id", householdId);`
- Return `NextResponse.json({ success: true })`.
- Wrap in try/catch: `log.error("POST /api/money/delete-data error", error); return NextResponse.json({ error: "Failed to delete" }, { status: 500 });`
