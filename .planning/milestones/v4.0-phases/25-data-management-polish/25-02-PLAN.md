**ExportTransactionsDialog** (`components/money/export-transactions-dialog.tsx`):
- "use client" component
- Uses `useTranslations("money.export")` for all strings
- Trigger: Button with Download icon (lucide-react) and text from i18n
- Dialog (shadcn/ui Dialog) containing:
  - Two date inputs for `date_from` and `date_to` (use shadcn/ui `Input` type="date" — simpler than Calendar+Popover for date range; the existing transaction filter bar uses similar inputs)
  - Export button with loading state (`isExporting` state)
  - On click: `fetch(/api/money/export?date_from=X&date_to=Y)`, check response.ok, create blob, trigger download via `URL.createObjectURL` + programmatic `<a>` click + `URL.revokeObjectURL`
  - Error: `toast.error(t("error"))` using sonner
  - Success: close dialog
- Props: none (self-contained, manages own state)

**DeleteMoneyDataDialog** (`components/money/delete-money-data-dialog.tsx`):
- "use client" component
- Uses `useTranslations("money.deleteData")` for all strings
- Trigger: Button with Trash2 icon (lucide-react), destructive variant
- AlertDialog (shadcn/ui AlertDialog) containing:
  - Warning message explaining what will be deleted (all transactions, budgets, bills, goals, net worth data, bank connections, household membership)
  - Text input where user must type "DELETE" exactly
  - Submit button disabled until input matches "DELETE"
  - Loading state (`isDeleting`)
  - On submit: `fetch("/api/money/delete-data", { method: "POST", body: JSON.stringify({ confirmation: "DELETE" }) })`
  - Success: `toast.success(t("success"))`, then `router.push("/money")` (redirect to money home which will show empty state via lazy household creation)
  - Error: `toast.error(t("error"))`
- Import `useRouter` from `next/navigation`
