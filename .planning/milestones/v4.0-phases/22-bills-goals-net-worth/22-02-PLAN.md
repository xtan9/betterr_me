  <files>
    app/api/money/bills/route.ts
    app/api/money/bills/[id]/route.ts
    app/api/money/bills/sync/route.ts
  </files>
  <action>
    **Bills list and create** (`app/api/money/bills/route.ts`):
    Follow the existing money API pattern (see `app/api/money/budgets/route.ts`):
    - `GET`: createClient -> getUser -> 401 if null -> resolveHousehold -> RecurringBillsDB.getByHousehold(householdId)
      - Return: `{ bills: RecurringBill[] }` -- include ALL bills (active, confirmed, dismissed) so UI can group them
      - Also compute and return summary: `{ bills, summary: { total_monthly_cents, bill_count, pending_count } }`
      - To compute total_monthly_cents: sum amount_cents for non-dismissed active bills, normalizing all frequencies to monthly (WEEKLY * ~4.33, BIWEEKLY * ~2.17, SEMI_MONTHLY * 2, MONTHLY * 1, ANNUALLY / 12). Use Math.round for integer result.
      - pending_count = bills with user_status === 'auto'
    - `POST`: createClient -> getUser -> 401 -> resolveHousehold -> validate body with billCreateSchema -> convert amount to negative cents (toCents(-body.amount)) since bills are expenses -> RecurringBillsDB.create({ household_id, name, amount_cents, frequency, next_due_date, source: 'manual', user_status: 'confirmed' })
      - Manual bills are auto-confirmed since user explicitly created them
      - Return 201 with created bill

    **Single bill operations** (`app/api/money/bills/[id]/route.ts`):
    - `PATCH`: createClient -> getUser -> 401 -> resolveHousehold -> validate body with billUpdateSchema -> RecurringBillsDB.update(id, updates)
      - If body includes `amount` (dollars), convert to negative cents
      - If body includes `user_status`, pass through directly
      - Return updated bill
    - `DELETE`: createClient -> getUser -> 401 -> RecurringBillsDB.delete(id)
      - Return 204 No Content

    **Plaid recurring sync** (`app/api/money/bills/sync/route.ts`):
    - `POST`: createClient -> getUser -> 401 -> resolveHousehold
    - Fetch all bank connections for the household via BankConnectionsDB
    - For each active connection: get access token from Vault (getAccessToken), call fetchRecurringTransactions(accessToken)
    - Map Plaid account_ids to our internal account UUIDs by querying accounts table
    - Collect all outflow_streams from all connections
    - Call RecurringBillsDB.upsertFromPlaid(householdId, detectedBills)
    - Return `{ synced: true, bills_found: count }`
    - Wrap in try/catch: log errors but don't fail on individual connection errors (some connections may have expired tokens)
  </action>

  <action>
    **Bills SWR hook** (`lib/hooks/use-bills.ts`):
    Follow existing hook pattern (see `lib/hooks/use-budgets.ts`):
    - `useBills()`: SWR hook fetching from `/api/money/bills`
    - Return `{ bills, summary, isLoading, error, mutate }`
    - Use `keepPreviousData: true` for smooth transitions
    - Derive `isLoading` from `!data && !error` (consistent with other hooks)

    **Sync pipeline integration** (`lib/plaid/sync.ts`):
    At the END of the existing `syncTransactions()` function (after transaction sync completes successfully):
    - Import `fetchRecurringTransactions` from `./recurring`
    - Import `RecurringBillsDB` from `@/lib/db`
    - After successful transaction sync, call `fetchRecurringTransactions(accessToken)` -- reuse the same access token that was already fetched for transaction sync
    - Map Plaid account_ids to internal UUIDs using the accounts already available in scope
    - Call `RecurringBillsDB(adminClient).upsertFromPlaid(householdId, outflowBills)`
    - Wrap in try/catch so recurring sync failure does NOT fail the transaction sync (log.warn and continue)
    - This ensures bills are refreshed every time transactions sync (daily cron + on-demand)

    **Net worth snapshot capture** (also in `lib/plaid/sync.ts`):
    AFTER the recurring bills sync block above, add a separate net worth snapshot block:
    - Import `NetWorthSnapshotsDB`, `ManualAssetsDB`, `MoneyAccountsDB` from `@/lib/db`
    - Import `getLocalDateString` from `@/lib/utils`
    - Compute current net worth:
      1. Fetch all accounts for the household: `MoneyAccountsDB(adminClient).getByHousehold(householdId)`
      2. Fetch manual assets: `ManualAssetsDB(adminClient).getByHousehold(householdId)`
      3. Sum: positive account balances + manual asset values = assets_cents, abs(negative account balances) = liabilities_cents, net = assets - liabilities
      4. Upsert snapshot: `NetWorthSnapshotsDB(adminClient).upsert(householdId, today, total, assets, liabilities)`
    - Use `getLocalDateString()` for today's date (consistent with project timezone convention)
    - Wrap in try/catch so snapshot failure does NOT fail the transaction sync (log.warn and continue)

    Note: The sync pipeline already has `adminClient` (createAdminClient) and the access token available. Both recurring bill sync and net worth snapshot piggyback on this existing infrastructure, each in independent try/catch blocks so failures are isolated.
  </action>
