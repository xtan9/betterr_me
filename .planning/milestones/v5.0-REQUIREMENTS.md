# Requirements: BetterR.Me v5.0 Fitness Tracking

**Defined:** 2026-02-23
**Core Value:** Users see accurate stats, the API rejects bad input, and the codebase is maintainable

## v5.0 Requirements

Requirements for v5.0 Fitness Tracking milestone. Each maps to roadmap phases.

### Exercise Library

- [x] **EXER-01**: User can browse a preset exercise library of 80-120 common exercises organized by muscle group and equipment
- [x] **EXER-02**: User can search exercises by name and filter by muscle group and equipment type
- [x] **EXER-03**: User can create custom exercises with name, primary muscle group, equipment, and exercise type
- [x] **EXER-04**: User can edit and delete their custom exercises
- [x] **EXER-05**: Each exercise has an exercise type that determines tracking fields (weight+reps, bodyweight+reps, or duration)

### Workout Logging

- [x] **WLOG-01**: User can start an empty workout session with elapsed time tracking
- [x] **WLOG-02**: User can add exercises to an active workout from the exercise library picker
- [x] **WLOG-03**: User can log sets with weight and reps for weight-based exercises
- [x] **WLOG-04**: User can log sets with reps only for bodyweight exercises (optional added weight)
- [x] **WLOG-05**: User can log sets with duration for timed exercises
- [x] **WLOG-06**: User can mark individual sets as complete with a checkbox
- [x] **WLOG-07**: User can finish or discard an active workout session
- [x] **WLOG-08**: Active workout persists across page refresh (dual-write to server + localStorage)
- [x] **WLOG-09**: User sees previous workout values for the same exercise auto-filled alongside set inputs
- [x] **WLOG-10**: User can label sets as warmup, normal, drop set, or failure
- [x] **WLOG-11**: User can add a title and freeform notes to a workout
- [x] **WLOG-12**: User can add notes to individual exercises within a workout

### Rest Timer

- [x] **REST-01**: Rest timer countdown auto-starts when a set is marked complete
- [x] **REST-02**: User can configure default rest timer duration (default 90s)
- [x] **REST-03**: User can adjust rest timer with +15s/-15s buttons during countdown
- [x] **REST-04**: Rest timer plays an audio beep on completion via Web Audio API
- [x] **REST-05**: Rest timer shows correct remaining time even after switching browser tabs

### Routines

- [x] **ROUT-01**: User can create a routine with a name and an ordered list of exercises with target sets/reps/weight
- [x] **ROUT-02**: User can edit and delete routines
- [x] **ROUT-03**: User can start a workout from a routine, which pre-fills exercises and targets (copy-on-start)
- [x] **ROUT-04**: User can save a completed workout as a new routine template

### History & Progression

- [x] **HIST-01**: User can view a reverse-chronological list of past workouts with summary info (date, title, exercises, volume, duration)
- [x] **HIST-02**: User can view a completed workout detail showing all exercises, sets, PRs achieved, and duration
- [x] **HIST-03**: User can view per-exercise progression charts showing weight/volume over time with date range selector (1m/3m/6m/all)
- [x] **HIST-04**: Personal records (max weight, best set volume) are automatically detected and stored per exercise
- [x] **HIST-05**: User sees a congratulatory PR banner mid-workout when a new personal record is set
- [x] **HIST-06**: Dashboard shows last workout date and current week's workout count

### Settings & Navigation

- [x] **SETT-01**: User can select weight unit preference (kg or lbs) in settings
- [x] **SETT-02**: All weight displays and inputs respect the user's unit preference (stored as kg internally)
- [x] **SETT-03**: Workouts page appears as a top-level sidebar nav item with Dumbbell icon

### i18n

- [x] **I18N-01**: All fitness tracking UI strings are translated in en, zh, and zh-TW

## Future Requirements

Deferred to v4.1+ milestone. Tracked but not in current roadmap.

### Deferred Features

- **DFRD-01**: Superset grouping (pair 2-3 exercises to perform back-to-back)
- **DFRD-02**: Muscle group distribution chart (weekly volume per muscle group)
- **DFRD-03**: Estimated 1RM as a PR type (requires formula selection)
- **DFRD-04**: Workout streak tracking (reuse habit streak pattern)
- **DFRD-05**: Complete exercise name translations in zh/zh-TW (ship UI strings first, exercise names incrementally)

## Out of Scope

Explicitly excluded. Documented to prevent scope creep.

| Feature | Reason |
|---------|--------|
| Social feed / sharing / leaderboards | Single-user app, would change product identity |
| AI workout generation | Requires LLM integration, premature for v5.0 |
| Cardio distance/pace/route tracking | Web can't access GPS; duration-only covers treadmill/bike adequately |
| Body measurement tracking | Separate product feature requiring camera/photo storage |
| Plate calculator | Users know their plate math; low value vs. effort |
| Video/GIF exercise demonstrations | Content licensing/hosting problem, not code |
| Live Activity / watch integration | Native-only platform APIs |
| Workout programs / periodization | Complex program engine (5/3/1, nSuns) is premature |
| RPE (Rate of Perceived Exertion) | Niche field, can add later as optional |
| Mobile app | Web-only for v5.0 |
