## Phase Details

### Phase 18: Database Foundation & Exercise Library
**Goal**: Users can browse, search, and manage exercises — and the schema that all future phases build on is locked
**Depends on**: Phase 17 (v3.0 complete)
**Requirements**: EXER-01, EXER-02, EXER-03, EXER-04, EXER-05, SETT-01, SETT-02, SETT-03
**Success Criteria** (what must be TRUE):
  1. User can browse ~80-120 preset exercises organized by muscle group and equipment, visible to all authenticated users
  2. User can search exercises by name and filter by muscle group or equipment type without any server round-trips on each keystroke
  3. User can create a custom exercise with a name, primary muscle group, equipment, and exercise type — and the exercise appears only for that user
  4. User can edit and delete their own custom exercises; preset exercises cannot be edited or deleted
  5. User can select kg or lbs in Settings and all weight displays and inputs throughout the app reflect the chosen unit
**Plans**: 4 plans
Plans:
- [x] 18-01-PLAN.md -- Database schema (6 tables), TypeScript types, fitness utilities, validation schemas
- [x] 18-02-PLAN.md -- Exercise CRUD (ExercisesDB, API routes, SWR hook, exercise library UI)
- [x] 18-03-PLAN.md -- Settings weight unit, sidebar nav, placeholder workouts page, i18n strings
- [ ] 18-04-PLAN.md -- Gap closure: mount ExerciseLibrary on /workouts/exercises page route

### Phase 19: Workout Logging Core Loop
**Goal**: Users can log a complete workout session end-to-end — and session state survives browser refresh
**Depends on**: Phase 18
**Requirements**: WLOG-01, WLOG-02, WLOG-03, WLOG-04, WLOG-05, WLOG-06, WLOG-07, WLOG-08, WLOG-09, WLOG-10, WLOG-11, WLOG-12, REST-01, REST-02, REST-03, REST-04, REST-05
**Success Criteria** (what must be TRUE):
  1. User can start a workout, add exercises from the library, log sets (weight+reps, reps-only, or duration per exercise type), mark sets complete, and finish — with the full session saved to the database
  2. User who refreshes mid-workout sees a "Resume workout?" banner and all previously logged sets are restored
  3. Rest timer auto-starts when a set is marked complete, counts down correctly after switching browser tabs, plays an audio beep at zero, and can be adjusted with +15s/-15s buttons
  4. User sees the previous workout's values for each exercise pre-filled alongside the set input fields as a reference
  5. User can label sets as warmup, normal, drop set, or failure and add a title, freeform notes to the workout, and notes per exercise
**Plans**: 7 plans
Plans:
- [x] 19-01-PLAN.md -- Schema migration (discarded status), WorkoutsDB, WorkoutExercisesDB, Zod validation schemas
- [x] 19-02-PLAN.md -- Workout API routes (start, active, update, nested exercise/set CRUD)
- [x] 19-03-PLAN.md -- Rest timer hook, audio beep, localStorage session helpers
- [x] 19-04-PLAN.md -- useActiveWorkout SWR hook, workout logger page, header, exercise cards, set rows
- [x] 19-05-PLAN.md -- Exercise picker sheet, rest timer UI, finish/discard dialogs, resume banner
- [x] 19-06-PLAN.md -- i18n translations for workout logging UI in en, zh, zh-TW
- [ ] 19-07-PLAN.md -- Gap closure: wire exercise picker sheet, finish/discard dialogs, rest timer edit

### Phase 20: Routines & Templates
**Goal**: Users can create reusable workout templates and start workouts from them
**Depends on**: Phase 19
**Requirements**: ROUT-01, ROUT-02, ROUT-03, ROUT-04
**Success Criteria** (what must be TRUE):
  1. User can create a routine with a name and an ordered list of exercises with target sets, reps, and weight
  2. User can edit and delete existing routines
  3. User can start a workout from a routine and the exercises and targets are pre-filled; editing the routine afterwards does not change the already-started workout
  4. User can save a completed workout as a new routine template from the workout detail view
**Plans**: 3 plans
Plans:
- [ ] 20-01-PLAN.md -- RoutinesDB class, Zod validation schemas, routine CRUD API routes
- [ ] 20-02-PLAN.md -- Copy-on-start + save-as-routine endpoints, useRoutines hook, routine UI components, routines page
- [ ] 20-03-PLAN.md -- Wire routines into workouts landing + finish dialog, i18n for all 3 locales

### Phase 21: History, Personal Records & Progression
**Goal**: Users can review their workout history, see personal records detected automatically, and track per-exercise progression over time
**Depends on**: Phase 20
**Requirements**: HIST-01, HIST-02, HIST-03, HIST-04, HIST-05, HIST-06, I18N-01
**Success Criteria** (what must be TRUE):
  1. User can view a reverse-chronological list of past workouts with date, title, exercise names, total volume, and duration
  2. User can view a completed workout detail showing every exercise, every set, which sets were PRs, and the total duration
  3. User sees a congratulatory PR banner mid-workout when the current set exceeds their stored personal record for that exercise
  4. User can view a per-exercise progression chart (max weight and volume over time) with a 1m/3m/6m/all date range selector
  5. Dashboard shows last workout date and current week's workout count alongside existing habits and tasks widgets
**Plans**: 4 plans
Plans:
- [ ] 21-01-PLAN.md — Data foundation: shadcn chart install, DB methods (getWorkoutsWithSummary, getExerciseHistory), personal-records utility, API routes, SWR hooks
- [ ] 21-02-PLAN.md — Dashboard workout stats widget (last workout date, weekly count) with i18n
- [ ] 21-03-PLAN.md — Workout history list UI, workout detail page (/workouts/[id]) with i18n
- [ ] 21-04-PLAN.md — Exercise progression chart (Recharts), PR banner mid-workout, PR detection integration with i18n
